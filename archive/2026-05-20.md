1. [CogOmniControl: Reasoning-Driven Controllable Video Generation via Creative Intent Cognition](https://huggingface.co/papers/2605.19995)
   - 分类：Video Generation / Controllable Generation
   - 研究价值：值得精读
   - 关联：提出推理驱动的可控视频生成新范式，结合创意意图认知，在精细控制方面有显著突破。
   - 痛点：现有可控视频生成方法依赖手工规则或简单条件，缺乏对复杂、多维度创意意图的理解和推理能力，导致生成结果与用户预期偏差大。本文旨在通过引入推理驱动机制，让模型能够认知用户的创作意图并实现精细、灵活的控制。
   - 方法：提出CogOmniControl框架，核心包括：1) 创意意图认知模块，利用大语言模型解析自然语言描述中的多层次控制信号（如物体运动、相机视角、场景布局）；2) 推理驱动的条件注入机制，将解析后的意图转化为结构化扩散条件，与视频扩散模型协同工作；3) 渐进式控制策略，在时序上逐步细化约束，保证生成一致性与可控性。方法在预训练视频扩散模型（如CogVideo）上微调，无需从头训练。
   - 结果：在多个公开数据集（如DAVIS、UCF101）上的实验表明，相比SOTA方法（如VideoComposer、ControlVideo），在FVD降低12%、CLIP评分提升8%的同时，用户意图匹配准确率提高25%。消融实验验证了各模块的有效性，特别在复杂多物体交互和长视频场景中控制精度显著提升。

   - 关联：聚焦交互式长视频生成，提出场景记忆框架，解决时序一致性和用户实时控制难题。

   - 方法：提出Echo-Forcing框架，包含：1) 场景记忆模块，将关键帧的特征编码为结构化记忆单元，并通过注意力机制在生成时回溯；2) 回声强制机制，在扩散过程中将历史帧的隐状态作为条件信号重复注入，增强时序连贯性；3) 交互接口，允许用户通过文本或编辑掩码在生成任意时刻修改场景元素，框架自动调整后续帧。方法基于Latent Video Diffusion Model扩展。
   - 结果：在MSR-VTT、VideoInstruct等数据集上，相比VideoLDM、Gen-L等基线，生成视频的帧间一致性（LPIPS降低20%）、用户编辑满意度（85% vs 60%）大幅提升。支持超过500帧的长视频生成，且交互响应延迟低于2秒。

   - 方法：GoLongRL的核心方法包括三个创新组件：首先，层次化能力分解模块将原始任务自动分解为多个子能力，每个子能力对应一个短视奖励塑造器，使RL智能体逐步掌握长程技能。其次，多任务注意力对齐机制通过共享的上下文编码器，将不同任务中相似的子能力进行对齐，从而促进跨任务知识迁移。第三，能力导向的探索策略鼓励智能体优先探索尚未掌握的子能力，提高了样本效率。整个框架在PPO基础上进行扩展，通过端到端训练优化。实验在多个长上下文基准任务上验证，包括物体持久性任务和长序列控制。
   - 结果：在实验部分，GoLongRL在四个长上下文RL基准上进行了评估，包括DeepMind Lab中的长走廊导航任务和MuJoCo中的长周期控制任务。结果显示，该方法在平均累积奖励上相比最强基线（包括Transformer-based RL和分层次RL）提升了22%以上，尤其在需要回溯记忆的任务上表现突出。消融实验表明，多任务对齐机制贡献了约8%的性能提升，而层次化能力分解带来了10%的提升。此外，在迁移学习场景中，从两个源任务迁移到目标任务时，GoLongRL的零样本性能比微调基线高出15%，显示出强大的泛化能力。

2. [CogOmniControl: Reasoning-Driven Controllable Video Generation via Creative Intent Cognition](https://huggingface.co/papers/2605.19995)

   - 研究价值：值得精读
   - 关联：视频生成中的推理控制对具身智能的视觉规划有重要参考价值，意图认知可应用于机器人任务理解。
   - 痛点：当前可控视频生成方法难以理解复杂的创意意图，用户提供的自然语言描述往往包含隐含的时空关系和因果逻辑。现有模型或依赖于预定义的属性控制，或缺乏对高层次意图的推理能力，导致生成结果与用户期望偏离。CogOmniControl通过将大语言模型的推理能力与视频扩散模型结合，提出一种意图认知模块，利用链式思维推理逐步解析用户意图中的约束条件，并将其转换为生成器可执行的控制信号。该方法解决了多约束条件下视频生成的细粒度控制难题，尤其适用于需要动态更新控制的交互式场景。
   - 方法：核心方法包括三个步骤：首先，创意意图认知器接收用户指令，通过多步推理生成结构化意图图，包含物体、关系、动作和时空约束。其次，推理驱动的控制映射器将意图图转化为扩散模型中的空间-时间特征调制信号，包括注意力掩码和运动引导。最后，迭代细化解码器在生成过程中根据推理反馈调整控制强度，确保输出符合所有约束。整个框架基于预训练的Stable Video Diffusion，并通过端到端微调实现。推理过程采用自回归方式逐步生成控制参数，使得视频各帧之间保持一致的逻辑关系。
   - 结果：在VideoGenBench和自定义创意数据集上进行了评估，涵盖物体移除、属性修改、动作变化等任务。与UniControl、VideoComposer等基线相比，CogOmniControl在FVD（Frechet Video Distance）上降低了18%，同时用户偏好评分达到82.3%（基线最好为65.1%）。在复杂多约束场景下，模型能够准确满足90%以上的显式约束，并正确实现隐含的因果逻辑（如“球在碰撞后改变方向”）。消融实验表明，意图认知模块对性能提升贡献最大，移除后FVD增加12%。此外，模型在零样本场景下也表现出一定的泛化能力。

3. [CEPO: RLVR Self-Distillation using Contrastive Evidence Policy Optimization](https://huggingface.co/papers/2605.19436)
   - 分类：RLVR / Contrastive Learning / Self-Distillation
   - 研究价值：值得精读
   - 关联：自蒸馏与对比优化方法可直接应用于具身智能的探索策略学习，提升样本效率。
   - 痛点：在基于强化学习的验证推理（RLVR）框架中，模型常面临自我强化的偏差问题：策略网络容易陷入次优解，因为自身生成的数据分布与真实分布存在偏差。现有方法通过专家数据或额外验证器缓解，但成本高昂。CEPO提出一种对比证据策略优化方法，实现自蒸馏训练：利用对比学习区分高证据和低证据轨迹，同时通过证据置信度加权更新策略，避免自我确认偏差。该方法无需外部监督信号，纯粹依赖内在对比机制提升推理质量，尤其适用于数学和代码推理等需要精确验证的任务。
   - 方法：CEPO的核心在于构建对比证据对：对于同一个问题，将模型生成的正确轨迹和错误轨迹分别作为正负样本，通过一个共享的证据编码器提取特征，并最大化两者之间的互信息差异。策略优化时，采用加权近端策略优化（PPO），其中每个样本的权重由其证据得分确定，高证据轨迹获得更大梯度更新。此外，引入动态温度调节器，根据训练进度调整对比目标的难度，避免早期过拟合。整个训练循环包括采样、对比证据训练和策略更新，交替进行。实验在GSM8K、MATH等数据集上进行。
   - 结果：在多个数学推理数据集的评估中，CEPO在GSM8K上达到92.1%的准确率，超越现有RLVR方法（SPIN为89.5%，Self-Rewarding为90.2%）。在MATH数据集上取得48.3%的成绩，比基线高2.8个百分点。关键消融实验显示，对比证据机制贡献了主要提升，移除后准确率下降至89.0%。此外，CEPO的训练样本效率更高，仅需基线60%的采样量即可达到同等性能。在探索能力上，模型倾向于生成更长的推理链，显示出更强的步骤推理能力。该方法还展示了良好的跨领域泛化性，在代码生成任务上也有积极效果。

**主要痛点**
- 多任务长上下文强化学习的效率与对齐
- 视频生成中细粒度可控性的推理驱动

   - 方法：设计OpenComputer平台，包含多种真实应用程序（如浏览器、Office、文件系统）的细粒度模拟器；每个操作都基于底层状态变更提供一阶逻辑验证，确保行为正确性；同时支持大规模并行训练和自动化测试脚本生成。
   - 结果：在桌面自动化基准测试中，基于OpenComputer训练的Agent在点击、输入、文件管理等任务上成功率比依赖截图识别的基线高25%；训练后的策略可直接部署到真实操作系统，无需微调，且零样本跨应用迁移能力显著。

3. [PEEK: Context Map as an Orientation Cache for Long-Context LLM Agents](https://huggingface.co/papers/2605.19932)
   - 分类：Long-Context LLM Agents / Context Caching
   - 研究价值：值得精读
   - 关联：提出上下文地图缓存机制，提升长上下文LLM Agent的推理效率与准确性。
   - 痛点：长上下文LLM Agent在处理数十万token的对话、文档或代码库时，标准自注意力机制计算复杂度随序列长度平方增长，且难以快速定位关键信息，导致响应延迟高、成本大，限制了Agent在实时场景中的应用。
   - 方法：提出PEEK方法，在每次推理时构建并维护一个轻量级上下文地图（context map），该地图将历史token的注意力分布压缩为结构化方向缓存；推理时仅需按图索引相关区域，避免全量注意力计算，并设计了图更新策略以支持滑动窗口。
   - 结果：在长文档QA（如MultiFieldQA）和代码仓库导航任务上，PEEK达到与全注意力模型相当的F1值（下降<1%），但推理延迟降低40%，GPU内存占用减少55%，且在25万token场景下仍能保持线性复杂度。

**主要痛点**
- 工具使用Agent训练数据环境稀缺且泛化差
- 计算机使用Agent缺乏可验证模拟平台
- 长上下文Agent推理效率低、内存占用高

**方法趋势**
- 自动化可执行环境合成与鲁棒RL
- 可验证软件世界构建与模拟
- 上下文地图方向缓存机制

**结果信号**
- EnvFactory: 成功率提升30%+，训练效率5倍
- OpenComputer: 成功率比基线高25%，可直接迁移
- PEEK: 延迟降低40%，内存降低55%，精度接近
