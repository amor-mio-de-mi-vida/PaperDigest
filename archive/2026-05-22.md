   - 方法：提出DecQ框架，在RAE解码器中引入轻量级细节浓缩查询（仅8个额外查询），通过冷凝模块从冻结VFM的中间层特征提取细粒度信息。这些查询与图像令牌一起输入解码器，并在扩散生成过程中被联合建模。该方法不修改编码器，仅增加3.9%计算开销，即可有效融合浅层和深层特征。

2. [The Value of Covariance Matching in Gaussian DDPMs and the Lanczos Sampler](https://arxiv.org/abs/2605.22723v1)
   - 分类：Gaussian DDPM / Covariance Matching / Sampler
   - 研究价值：值得精读
   - 关联：DDPM采样中协方差匹配的理论与算法改进
   - 痛点：高斯DDPM常用各向同性反向协方差近似，导致路径空间KL散度随步数T呈Ω(1/T)误差，限制了快速采样的保真度。完全匹配后验协方差可降低误差但需要计算和存储全协方差矩阵，计算成本极高。
   - 方法：理论证明全协方差匹配可将路径KL误差降至O(1/T^2)，实现阶数改善。提出Lanczos高斯采样器（LGS），一种无需训练、无需存储协方差矩阵的方法，仅通过均值网络的雅可比-向量积（JVP）进行协方差-向量乘，并利用Lanczos迭代近似从最优协方差采样。证明LGS近似误差随Lanczos步数指数衰减。
   - 结果：在CIFAR-10、ImageNet等基准上，仅使用3步Lanczos迭代即可超越强对角线协方差基线（如OCM-DDPM）的样本质量，几乎无额外计算开销。实验验证了协方差匹配的理论优势，并提供了实用的采样算法。

1. [Gated DeltaNet-2: Decoupling Erase and Write in Linear Attention](https://arxiv.org/abs/2605.22791v1)
   - 分类：Linear Attention / Efficient Transformer
   - 研究价值：值得精读
   - 关联：直接改进线性注意力机制，提升训练和推理效率，尤其在长上下文场景中表现突出。
   - 痛点：现有线性注意力模型（如Delta规则、Kimi Delta Attention）使用单标量门控同时控制擦除和写入，导致模型难以在保留旧知识的同时有效写入新信息，限制了长上下文任务的记忆控制精度和性能。
   - 方法：提出Gated DeltaNet-2，解耦擦除门（bt）和写入门（wt）为独立通道级门控向量，并保留通道级衰减；推导快速权重更新视角、分块WY算法和门控感知反向传播，支持高效并行训练。
   - 结果：在1.3B参数、100B FineWeb-Edu tokens训练下，Gated DeltaNet-2在语言建模、常识推理和检索任务上超越Mamba-2、Gated DeltaNet、KDA和Mamba-3；在长上下文RULER基准中多键检索设置提升显著，混合设置下保持优势。

2. [DeltaBox: Scaling Stateful AI Agents with Millisecond-Level Sandbox Checkpoint/Rollback](https://arxiv.org/abs/2605.22781v1)
   - 分类：Agent Runtime / State Management
   - 研究价值：值得精读
   - 关联：解决AI agent中状态检查点/回滚的延迟瓶颈，直接提升推理时搜索和强化学习的效率。
   - 痛点：AI agent在测试时树搜索或强化学习中需要频繁进行状态检查点与回滚，但现有方法需完整复制沙箱状态（文件+进程），导致百毫秒到秒级延迟，严重限制搜索深度和规模。
   - 方法：提出DeltaState抽象，通过追踪变更实现事务级C/R。DeltaFS：基于层的文件系统，冻结可写层并插入新层，回滚仅切换层；DeltaCR：增量转储进程状态，并直接从冻结模板进程fork()加速回滚。构建DeltaBox沙箱。
   - 结果：DeltaBox实现检查点14ms、回滚5ms的毫秒级延迟。在SWE-bench和RL微基准上，固定时间预算下agent可探索显著更多节点，证明性能大幅提升。

3. [Vector Policy Optimization: Training for Diversity Improves Test-Time Search](https://arxiv.org/abs/2605.22817v1)
   - 分类：RL for LLM / Test-Time Search
   - 研究价值：值得精读
   - 关联：提出训练多样性的新范式，提升推理时搜索效率，与高效推理中的scaling策略密切相关。

   - 方法：提出Vector Policy Optimization (VPO)，利用向量值奖励（如多测试用例通过率）训练LLM输出一组解，每个解专门针对向量奖励空间中不同折衷。VPO作为GRPO优势估计器的即插即用替代。
   - 结果：在四个任务上，VPO在测试时搜索（pass@k, best@k）上匹配或超越标量RL基线，且随搜索预算增大差距扩大。在进化搜索设置中，VPO模型解决了GRPO模型完全无法解决的问题。

**主要痛点**

- LLM后训练缺乏多样性，不适应测试时搜索

**方法趋势**

**结果信号**
