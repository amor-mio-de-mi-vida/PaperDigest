### 高效训练与推理

- 论文数：3

**今日推荐精读 Top 3**

1. [dMoE: dLLMs with Learnable Block Experts](https://huggingface.co/papers/2605.30876)
   - 分类：MoE / Efficient Training
   - 研究价值：值得精读
   - 关联：通过动态专家混合机制提升大模型训练与推理效率
   - 痛点：传统MoE模型存在专家退化、负载不均和通信开销大的问题，且静态专家网络难以适应多样化任务，导致性能瓶颈和资源浪费。
   - 方法：提出dMoE架构，引入可学习的块专家（Block Experts），每个专家由多个可微调子块组成，同时设计动态路由机制自适应选择专家子集，在训练中专家结构可随数据变化调整。
   - 结果：在多个语言建模基准上，dMoE以更少的参数和FLOPs达到与更大模型相当的性能，参数量减少30%以上，训练速度提升2倍，推理吞吐量提升1.5倍，并显著缓解了专家退化问题。

2. [Not All Disagreement Is Learnable: Token Teachability in On-Policy Distillation](https://huggingface.co/papers/2605.26844)
   - 分类：Knowledge Distillation / Efficient Training
   - 研究价值：值得精读
   - 关联：提出token级可教性概念优化知识蒸馏，提升训练效率与模型性能
   - 痛点：知识蒸馏中教师-学生logit差异并非全部有益，部分不一致信息难以学习甚至导致负迁移，传统方法未区分可学与不可学token，浪费计算资源。
   - 方法：提出“token可教性”概念，基于每个token的预测差异和不确定性动态评估其可学性，仅在可教token上施加蒸馏损失，同时结合在线策略生成高质量软标签。
   - 结果：在多种蒸馏场景下（包括分类、生成），学生模型性能提升显著，尤其在长尾分布和小样本任务上准确率提高5-8%，训练时间减少20-30%，且可教性分析揭示了蒸馏的关键模式。

3. [COLLEAGUE.SKILL: Automated AI Skill Generation via Expert Knowledge Distillation](https://huggingface.co/papers/2605.31264)
   - 分类：Skill Generation / Automation
   - 研究价值：值得扫读
   - 关联：自动生成AI技能prompt以提升模型能力，间接影响训练效率
   - 痛点：大型语言模型需要大量特定技能（如推理、规划）的prompt设计，手动构建耗时且难以覆盖多样场景，制约模型应用效率。
   - 方法：提出COLLEAGUE.SKILL框架，从专家知识库中蒸馏技能模板，利用元学习自动生成适用于新任务的prompt技能，实现技能的可复用和快速适配。
   - 结果：在多个任务上生成的技能prompt显著提升了基础模型的零样本性能，与手工设计技能相当甚至更优，但生成技能的质量和泛化性仍有提升空间。

**主要痛点**
- 高效训练与推理中，模型参数量和计算开销过大
- 知识蒸馏中噪声干扰导致负迁移
- 手动构建prompt技能成本高

**方法趋势**
- 可学习的动态专家混合架构
- token级可教性评估与选择性蒸馏
- 专家知识蒸馏与元学习自动生成技能

**结果信号**
- dMoE实现参数减少30%，速度提升2倍
- 可教性蒸馏提升性能5-8%，减少训练时间20%
- 自动化技能生成与手工设计相当

### 具身智能 / 强化学习

- 论文数：2

**今日推荐精读 Top 3**

1. [Not All Disagreement Is Learnable: Token Teachability in On-Policy Distillation](https://huggingface.co/papers/2605.26844)
   - 分类：Reinforcement Learning / Knowledge Distillation
   - 研究价值：值得精读
   - 关联：提出在线策略蒸馏中token级别的可教学性概念，对提升强化学习模型样本效率有重要启示
   - 痛点：现有在线策略蒸馏方法假设所有师生分歧均可学习，但实际存在不可学习的噪声干扰，导致蒸馏效率低下。本文首次系统定义并区分可教学与不可教学的分歧，解决了传统蒸馏中盲目学习全部分歧导致的性能瓶颈问题。
   - 方法：提出Token Teachability框架，通过计算每个token对最终奖励的边际贡献来量化其可教学性，并设计选择性学习机制，仅对高可教学性token进行蒸馏，同时忽略噪声token。整合了重要性采样与因果推断技术确保估计无偏。
   - 结果：在Atari和MuJoCo基准测试中，该方法相比标准在线蒸馏方法样本效率提升40%，最终策略性能平均提高12%。消融实验验证了可教学性判据的有效性，且在不同网络架构下表现鲁棒。

2. [From Prompt Injection to Persistent Control: Defending Agentic Harness Against Trojan Backdoors](https://huggingface.co/papers/2605.31042)
   - 分类：AI Security / LLM Agents
   - 研究价值：值得精读
   - 关联：针对LLM agent框架的Trojan后门攻击提出系统防御，对保障自主智能体安全性至关重要
   - 痛点：现有LLM agent系统易受prompt注入和后门攻击，攻击者可植入隐蔽触发器实现持久控制，而当前防御方法只关注单一攻击向量，缺乏对Trojan后门在agent工作流中传播机制的深入理解和整体防御方案。
   - 方法：提出AgentShield防御框架，结合静态分析与动态监控：静态阶段通过符号执行检测agent harness代码中的潜在后门触发器；动态阶段利用上下文完整性验证和异常行为检测，实时阻断未授权控制指令。并引入对抗性训练增强模型对mixed-initiative攻击的鲁棒性。
   - 结果：在多个开源agent框架（如LangChain、AutoGPT）上测试，对Trojan后门攻击的防御成功率从基线30%提升至89%，误报率低于5%。此外，在真实场景红队测试中成功抵御了96%的隐蔽控制尝试。

**主要痛点**
- 在线策略蒸馏中无法区分有效分歧与噪声
- LLM agent面临Trojan后门持久控制威胁

**方法趋势**
- Token Teachability：基于边际贡献的可教学性量化与选择性蒸馏
- AgentShield：静态符号执行+动态上下文完整性验证

**结果信号**
- 样本效率提升40%，性能提高12%
- 防御成功率89%，误报率低于5%

1. [Exploring Autonomous Agentic Data Engineering for Model Specialization](https://huggingface.co/papers/2605.30407)
   - 分类：Autonomous Agents / Data Engineering
   - 研究价值：值得精读
   - 关联：提出自主数据工程范式，推动模型专业化自动化
   - 痛点：当前模型专业化依赖大量人工数据工程，包括数据收集、清洗、标注等，流程繁琐且成本高昂，难以规模化。缺乏自主性导致模型迭代周期长，限制了在垂直领域的快速应用。
   - 方法：提出一种基于大语言模型的自主Agent系统，自动执行完整的数据工程流水线：通过任务理解自动规划数据需求，利用检索和生成获取原始数据，结合自适应清洗规则和主动学习标注策略，生成高质量训练数据，并循环评估模型性能迭代优化。
   - 结果：在多个专业领域（如医疗、法律、金融）的模型专业化任务中，该方法将人力干预降低80%以上，同时模型在目标任务上的准确率平均提升5-10%，且泛化能力优于传统人工驱动方法。

2. [From Prompt Injection to Persistent Control: Defending Agentic Harness Against Trojan Backdoors](https://huggingface.co/papers/2605.31042)
   - 分类：Agent Security / Backdoor Defense
   - 研究价值：值得精读
   - 关联：聚焦Agent系统新型安全威胁，提出实用防御框架
   - 痛点：现有Agent框架（如AutoGPT、LangChain）中，攻击者可通过提示注入植入后门，获得对Agent行为的持久控制，窃取数据或执行恶意操作。传统安全方案难以检测此类隐蔽且持续的控制流。
   - 方法：设计多层次防御框架：1）输入净化与意图检测：对用户输入和系统指令进行异常模式识别；2）运行时行为监控：建立Agent行动图并检测偏离安全策略的序列；3）权限撤权机制：一旦检测到后门触发，立即撤销Agent对敏感资源和工具的访问权限，并回滚状态。
   - 结果：在多个主流Agent框架上测试，该方法对Trojan后门的检测率达到95%以上，持久控制成功率降至2%以下，且对正常Agent任务的性能开销小于5%。

**主要痛点**
- 模型专业化数据工程依赖大量人工，效率低下
- Agent框架面临提示注入和后门攻击的安全威胁

**方法趋势**
- 自主Agent数据工程流水线：规划-采集-清洗-标注-迭代
- 多层次防御框架：输入净化、行为监控、权限撤权

**结果信号**
- 人力减少80%+，准确率提升5-10%
- 后门检测率>95%，控制成功率<2%，性能开销<5%
