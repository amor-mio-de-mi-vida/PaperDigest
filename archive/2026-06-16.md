1. [Exact Posterior Score Estimation for Solving Linear Inverse Problems](https://arxiv.org/abs/2606.17048v1)
   - 分类：Diffusion / Linear Inverse Problems
   - 研究价值：值得精读
   - 关联：精确后验分数估计，解决了扩散模型用于逆问题的核心理论问题
   - 痛点：现有的扩散模型在解决线性逆问题时，通常使用基于近似似然修正或训练专用条件模型的方法，无法直接利用预训练无条件扩散模型进行精确后验采样。本文通过理论推导，得到了高斯线性逆问题下后验分数的闭式表达式，揭示了后验采样等价于在特定噪声协方差下的去噪问题，从而实现了无需梯度或投影的精确后验采样，解决了这一核心理论痛点。
   - 方法：通过推导得到后验分数的精确形式，并转化为去噪训练目标（Exact Posterior Score, EPS），该目标保留了标准预训练模型的输入输出结构，支持从零训练或从预训练去噪器微调。训练时使用与基础扩散模型相同的噪声分布，但引入与算子相关的偏移和协方差，使得模型学习校正后的后验分数。推理时直接应用标准采样器，无需任何启发式修正或额外似然梯度。
   - 结果：在FFHQ和ImageNet上五个线性逆任务（超分辨率、去模糊、修补等）中，EPS在保真度（PSNR、SSIM）、感知质量（FID、LPIPS）和分布度量上均优于无训练（DPS、MCG）和有训练（RedDiff）基线，且去噪器评估次数减少约一个数量级（从1000步降至100步），显著提升推理效率。

2. [LESS Is More: Mutual-Stability Sampling for Diffusion Language Models](https://arxiv.org/abs/2606.16908v1)
   - 分类：Diffusion Language Model / Adaptive Sampling
   - 研究价值：值得精读
   - 关联：针对扩散语言模型推理效率的有效自适应采样方法
   - 痛点：扩散语言模型（dLLMs）的推理通常使用固定步数的反向去噪，导致计算资源浪费在已经稳定的位置上，且可能过早决定不稳定的位置。缺乏一种训练无关、模型无关的自适应采样策略来动态决定每个标记的提交时机，现有自适应方法往往依赖额外训练或特定模型结构。
   - 方法：提出LESS自适应采样器，基于互稳定规则联合考虑预测置信度、标记持久性和分布稳定性（通过top-K Jensen-Shannon散度）。将标记提交视为在线停止问题，只在满足所有稳定性条件时才揭晓掩码位置，从而动态调整每个位置的反向步数。该方法是训练无关的，只需在推理时计算额外少量度量。
   - 结果：在Dream-7B、LLaDA-8B等模型上，LESS相比强训练无关自适应采样器（如动态截止）提升了平均准确率（在7个基准上平均提高1.2%），同时减少了72.1%的反向步骤，显著降低推理延迟和计算量，单步减少等价于减少变压器前向传播次数。

3. [DreamX-World 1.0: A General-Purpose Interactive World Model](https://arxiv.org/abs/2606.16993v1)
   - 分类：Interactive World Model / Video Generation
   - 研究价值：值得精读
   - 关联：通用交互式世界模型，支持多种控制，高效推理
   - 痛点：现有视频世界模型难以同时支持精确相机控制、场景记忆保持、长时程生成及事件组合控制。同时，在推理速度上难以达到实时交互需求，通常需要大量计算资源且存在颜色和风格漂移问题。
   - 方法：提出DreamX-World 1.0，集成多种创新：E-PRoPE实现高效相机控制（投影位置编码的轻量变体）；记忆条件场景持久性通过相机几何检索保持长程一致性；事件指令微调组合事件控制；强化学习对齐提升质量；混合精度执行、残差重用、75%剪枝VAE解码和异步流水线优化达到16 FPS。
   - 结果：在5秒基础评估中，相机控制得分73.75，总体得分84.76，优于HY-WorldPlay 1.5（80.79）和LingBot-World（80.45）。支持写实、游戏、风格化域，长时程生成中颜色漂移显著减少，且能在8×RTX 5090上实时运行。

**主要痛点**
- 扩散模型逆问题求解精度低、计算成本高
- 扩散语言模型推理固定步数浪费计算
- 世界模型缺乏精确控制和实时性

**方法趋势**

- 自适应互稳定采样
- 交互式世界模型集成多种优化

**结果信号**
- 逆问题性能提升且步数减少
- 语言模型推理效率显著提高
- 世界模型实现高质量实时交互

1. [KVEraser: Learning to Steer KV Cache for Efficient Localized Context Erasing](https://arxiv.org/abs/2606.17034v1)
   - 分类：LLM Inference / KV Cache
   - 研究价值：值得精读
   - 关联：高效训练与推理领域，KV缓存编辑直接降低推理延迟，适用于长上下文场景。
   - 痛点：大语言模型长上下文推理中，删除已处理文本段落后必须重算所有后续token，导致延迟随后缀长度线性增长。现有近似擦除方法无法保证全局一致性，精确擦除计算成本过高。痛点在于如何在不重算后缀的前提下实现局部上下文擦除，同时保持模型输出质量。
   - 方法：提出KVEraser学习型KV缓存编辑方法，仅替换被删除区间的KV状态为学习到的引导状态，其余缓存不变。两阶段训练：通用跨度邻居预训练学习擦除机制，任务特定微调适配下游。核心是训练一个轻量擦除器，通过监督学习使替换后的缓存状态近似于全重算结果。
   - 结果：在1K-32K上下文长度下，KVEraser擦除后性能接近全重算（几乎无损失），延迟仅增加24%（全重算增加17.6倍）。在未见过的长文档QA任务中，相比近似基线取得最佳性能，速度提升3-4倍。

2. [TokenPilot: Cache-Efficient Context Management for LLM Agents](https://arxiv.org/abs/2606.17016v1)
   - 分类：LLM Agent / Context Management
   - 研究价值：值得精读
   - 关联：高效训练与推理领域，针对Agent长会话的缓存优化，降低推理成本。
   - 痛点：LLM Agent在长时段会话中上下文不断累积，增加推理成本。现有文本修剪或动态记忆淘汰方法会破坏前缀结构，导致缓存失效（前缀不匹配），即文本稀疏性与缓存连续性之间存在关键权衡。痛点是如何在减少token足迹的同时维持前缀缓存连续性。
   - 方法：提出TokenPilot双粒度上下文管理框架：全局层通过摄入感知压缩稳定前缀并过滤环境噪声；局部层通过生命周期感知淘汰监控上下文段剩余效用，在任务相关性过期时保守地卸载内容。两种粒度协同工作，避免缓存失效。
   - 结果：在PinchBench和Claw-Eval上，隔离模式成本降低61%和56%，连续模式降低61%和87%，同时性能与先前系统持平。已集成到LightMem2中。

3. [ActiveSAM: Image-Conditional Class Pruning for Fast and Accurate Open-Vocabulary Segmentation](https://arxiv.org/abs/2606.16996v1)
   - 分类：Efficient Inference / Segmentation
   - 研究价值：值得精读
   - 关联：高效训练与推理领域，通过对类别进行条件剪枝加速分割推理。
   - 痛点：SAM 3等全景分割模型在全词汇表上运行全分辨率解码，但每张图像只包含少量活跃类别，导致大量不必要的计算。痛点是如何在零样本、无训练的前提下根据图像内容动态激活相关类别，避免全局解码开销。
   - 方法：提出ActiveSAM零样本框架：先规范化并扩展类别提示，然后从低分辨率预览中估计图像条件活跃集，仅对保留类别进行全分辨率解码，并采用带边距的背景校准抑制低置信度像素。整个流程无需训练、权重更新或真实类别标签。
   - 结果：在8个开放词汇分割基准上，平均mIoU提升约1.4，同时在大词汇数据集上速度提升高达5.5倍。在图像损坏模拟分布偏移下鲁棒性最强，适用于自动驾驶等场景。

**主要痛点**
- LLM推理中长上下文缓存管理导致高延迟和成本
- RL训练中稀疏奖励信号和低效数据利用
- 模型训练优化算法收敛慢或计算成本高

**方法趋势**
- KV缓存编辑（KVEraser）
- 双粒度上下文管理（TokenPilot）
- 图像条件类别剪枝（ActiveSAM）
- 基于证据树的规则监督（DeepRubric）
- 探索性RL利用奖励支架（ExpRL）

**结果信号**
- 推理延迟降低3-4倍，性能保持
- RL训练成本降低13倍，性能相当
- 分割速度提升5.5倍，精度提升1.4 mIoU
- 训练优化收敛更快，性能超越现有方法

1. [T-Rex: Tactile-Reactive Dexterous Manipulation](https://arxiv.org/abs/2606.17055v1)
   - 分类：Tactile Manipulation / Reinforcement Learning
   - 研究价值：值得精读
   - 关联：该论文直接解决具身智能中触觉反馈缺失的痛点，推动灵巧操作前沿。
   - 痛点：现有视觉-语言-动作模型在机器人操作中普遍忽略触觉模态，或仅使用静态触觉编码器，导致机器人无法在接触时动态调整策略。此外，缺乏大规模触觉训练数据和评估基准，以及当前VLA架构对高频触觉信号处理不足，严重限制了触觉反应性敏捷操作的发展，尤其在需要精细力控制和可变形物体操作的任务中表现不佳。
   - 方法：本文通过三方面创新突破：1）提出数据高效采集策略，优先收集100小时基本运动基元触觉数据；2）设计可变速率Mixture-of-Transformers (MoT)架构，通过门控混合不同速率注意力，并结合新的时序触觉VQ-VAE编码器，有效融合高频触觉信号而不牺牲现有VLA能力；3）在12个需精细力控制的任务上训练和评估。
   - 结果：在12个灵巧操作任务上，T-Rex平均成功率达到84.2%，而最强基线（不含触觉或使用静态触觉）平均成功率为64.7%，绝对提升19.5个百分点（相对30%）。具体任务如插销、线缆插入等，T-Rex均显著领先，证明触觉反馈对精细操作至关重要。

2. [Human Universal Grasping](https://arxiv.org/abs/2606.17054v1)
   - 分类：Grasping / Flow Matching
   - 研究价值：值得精读
   - 关联：利用人类抓取先验解决机器人抓取泛化问题，是具身智能中数据驱动方法的范例。
   - 痛点：多指机器人抓取仍远不及人类，主要瓶颈在于缺乏真实、多样、大规模的人类抓取数据来指导机器人学习。现有方法大多依赖合成数据或少量标注，泛化到新物体时表现差。此外缺少标准化评估基准和度量一致的3D模型，难以公平比较不同方法。人类每天执行数千次抓取，但机器人无法从这一丰富数据源学习。
   - 方法：构建1M-HUGs数据集：通过智能眼镜在41栋建筑内收集26小时自我中心抓取视频，涵盖6707个物体。提出HUG模型，采用流匹配从单张RGB-D图像生成抓取参数（手腕位姿+MANO手部姿态），并通过运动学重定向部署到机器人手。同时构建HUG-Bench，含90个来自5种几何类别的陌生物体及度量3D网格。
   - 结果：在HUG-Bench的30个物体真实测试中，HUG成功率达76%，比最强基线GraspNet的53%提升23%，比另一基线提升34%。在仿真中同样显著领先。HUG生成的抓取多样性高，且在不同机器人平台和家庭环境中表现强健，实现零样本泛化。

   - 痛点：当前通用机器人策略如VLA和世界-动作模型主要在2D图像帧或2D潜在空间操作，隐式处理3D几何，导致在接触丰富的操作任务中空间泛化能力不足。这些模型依赖大规模数据学习几何关系，却未显式利用几何先验，使得策略对物体姿态、工具方向等变化敏感，且模型往往笨重。

**主要痛点**

- 策略学习隐式处理几何先验

- 人类自我中心抓取数据与流匹配
- 几何基础模型复用与因果预测

- 乐观值估计与跨具身视频监督

1. [ExpRL: Exploratory RL for LLM Mid-Training](https://arxiv.org/abs/2606.17024v1)
   - 分类：RL for LLM / Mid-Training
   - 研究价值：值得精读
   - 关联：提出了基于参考解答的密级奖励信号，替代稀疏奖励进行中训练，与Agent的RL训练方法紧密相关
   - 痛点：针对LLM推理能力的中训练通常需要人工设计推理轨迹或依赖稀疏奖励RL，但稀疏奖励难以捕捉部分进展和有用中间步骤，限制了复杂推理任务的训练效率。手动设计轨迹成本高且可能不完整。
   - 方法：提出ExpRL方法，使用人类编写的问答数据作为奖励支架：隐藏参考解答，仅用于构建问题特定的评分标准，对策略采样的推理轨迹进行过程级或结果级密级奖励，从而强化部分正确行为和中间推理步骤，而非仅用最终答案奖励。
   - 结果：在挑战性数学推理任务上，ExpRL比SFT、稀疏奖励GRPO和自蒸馏产生更强的RL启动效果，并为后续稀疏奖励RL提供更好的初始化。混合领域实验表明该方法可扩展到数学以外的领域。

2. [TokenPilot: Cache-Efficient Context Management for LLM Agents](https://arxiv.org/abs/2606.17016v1)
   - 分类：LLM Agent / Context Management
   - 研究价值：值得精读
   - 关联：针对长会话Agent的上下文管理问题，提出双粒度管理框架，显著降低推理成本，实用价值高
   - 痛点：LLM Agent在长周期会话中上下文不断累积，导致推理成本急剧上升。现有文本修剪或动态内存驱逐方法虽能减少token，但会破坏前缀连续性，引起前缀不匹配和缓存失效，在成本与缓存连续性之间存在关键权衡。
   - 方法：提出TokenPilot双粒度上下文管理框架：全局层面，摄入感知压实（Ingestion-Aware Compaction）在摄入门控处稳定前缀并消除环境噪声；局部层面，生命周期感知驱逐（Lifecycle-Aware Eviction）监控上下文片段的剩余效用，采用保守的批次轮次调度，仅在任务相关性过期后才卸载内容。
   - 结果：在PinchBench和Claw-Eval上，孤立和连续模式下分别降低61%和56%的成本（孤立模式）以及61%和87%的成本（连续模式），同时保持有竞争力的任务性能。已集成到LightMem2开源库。

3. [Context-Aware RL for Agentic and Multimodal LLMs](https://arxiv.org/abs/2606.17053v1)
   - 分类：RL for LLM / Multimodal Agent
   - 研究价值：值得扫读
   - 关联：与Agent长程推理和多模态理解相关
   - 痛点：LLM在长上下文或复杂场景中难以定位关键证据，现有RL仅监督最终答案，缺乏细粒度的上下文理解训练。
   - 方法：提出ContextRL方法，包含一个间接的辅助RL目标：给定查询、答案和两个高度相似的上下文，模型需选择支持该查询-答案对的上下文，从而鼓励细粒度证据定位。通过条件过滤和生成式编辑构建对比上下文数据。
   - 结果：在5个长程基准上平均+2.2%，12个VQA基准上+1.8%，且对比实验证明增益来自上下文选择目标本身而非额外数据。

**主要痛点**
- 深度研究Agent的RL训练评分标准生成不完整
- 稀疏奖励RL难以捕捉中间步骤进展
- 长会话Agent上下文累积导致高成本且缓存连续性被破坏
- Agent证据筛选中LLM无法可靠区分符合/不符合纳入标准的研究

**方法趋势**
- 反向构建证据树-评分标准对（DeepRubric）
- 基于参考解答构建密级奖励信号（ExpRL）
- 双粒度上下文管理：摄入压实+生命周期驱逐（TokenPilot）
- 对比上下文选择RL目标（ContextRL）
