1. [JanusMesh: Fast and Zero-Shot 3D Visual Illusion Generation via Cross-Space Denoising](https://arxiv.org/abs/2606.20563v1)
   - 分类：3D Generation / Visual Illusion
   - 研究价值：值得精读
   - 关联：与3D生成领域紧密相关，解决了现有方法速度慢、几何不一致的问题。
   - 痛点：现有优化方法生成3D视觉幻觉速度慢（数小时）、颜色过饱和，且简单拼接方法会产生明显接缝和语义泄露，无法生成几何连贯的物体。
   - 方法：提出跨空间双分支去噪过程，动态解码3D潜变量到体素空间进行CLIP导向的视向对齐和有符号距离场(SDF)混合，确保无缝几何融合；再引入视图条件纹理合成模块，将2D扩散先验投影到融合几何上。
   - 结果：仅需3-5分钟即可生成高度真实、双语义的3D幻觉，在几何完整性、语义可识别性和效率上显著优于现有方法，实现零样本生成。

2. [SSD: Spatially Speculative Decoding Accelerates Autoregressive Image Generation](https://arxiv.org/abs/2606.20543v1)
   - 分类：Image Generation / Autoregressive Models
   - 研究价值：值得精读
   - 关联：针对自回归图像生成效率瓶颈，提出利用2D空间局部性的投机解码框架。
   - 痛点：自回归模型将图像展平为1D序列，丢失了视觉信号的2D空间局部性，导致推理时计算瓶颈严重，生成速度慢。
   - 方法：提出空间投机解码(SSD)，模型同时预测水平相邻和下方相邻两个token，利用2D空间相关性进行并行解码，突破视觉推理中的内存墙。
   - 结果：在DPG-Bench和GenEval上，自回归图像生成加速最高达13.3倍，同时保持高保真度，表明尊重视觉几何可释放巨大计算效率。

3. [The FID Lottery: Quantifying Hidden Randomness in Generative-Model Evaluation](https://arxiv.org/abs/2606.20536v1)
   - 分类：Evaluation / Metrics
   - 研究价值：值得精读
   - 关联：挑战FID作为评估指标的可靠性，对生成模型实验设计有重要指导意义。
   - 痛点：FID是图像生成的标准指标，但通常只报告单次训练+单次采样的数值，忽略了训练随机性和采样随机性带来的方差，导致评估结论不可靠。
   - 方法：将FID视为训练种子和生成种子双轴随机变量，在数百个SiT网络上大规模测量方差，并分析训练初始化、数据顺序、流匹配损失中每步高斯噪声等影响因素。
   - 结果：发现重训练导致FID波动比重采样大3.2倍；增大计算量或模型规模几乎不缩小波动（变异系数1-2%）；提出新评估协议：在每元最优引导下评估，任何低于~1.3%变异系数的FID差距视为不确定，并报告多个训练种子的误差棒。

**主要痛点**
- 3D视觉幻觉生成速度慢、几何不连贯
- 自回归图像生成因1D序列化导致计算瓶颈
- FID评估受随机性影响、结论不可靠
- 世界模型缺乏未观测时的状态持续演进能力
- 扩散模型时间步嵌入可能冗余

**方法趋势**
- 跨空间双分支去噪与SDF混合
- 空间投机解码利用2D相关性
- 大规模FID方差测量与统计协议
- WRBench基准与持久状态核分析
- 无时间步嵌入的理论与实验验证

**结果信号**
- JanusMesh 3-5分钟生成高质量3D幻觉
- SSD加速自回归生成达13.3倍
- FID变异系数1-2%，建议报告误差棒
- 现有世界模型均无法保持状态持续性
- 去除时间步嵌入FID不降反升

1. [UltraQuant: 4-bit KV Caching for Context-Heavy Agents](https://arxiv.org/abs/2606.20474v1)
   - 分类：KV Cache Quantization / Efficient Inference
   - 研究价值：值得精读
   - 关联：针对上下文密集的agent场景提出4-bit KV缓存压缩与部署优化，显著降低延迟并提升吞吐量。
   - 痛点：现有KV缓存压缩方法在上下文密集的agent场景中面临质量与效率的权衡，且缺乏针对多轮agent任务联合优化。长前缀复用和并发性导致GPU资源利用率低，而4-bit量化通常伴随精度损失，无法直接部署。
   - 方法：采用TurboQuant风格的旋转与码书量化作为质量锚点，结合vLLM FP8 KV缓存作为部署锚点。提出非对称K/V处理、Walsh-Hadamard旋转、QJL去除和块缩放变体。在AMD GPU上实现FP4近似路径，使用FP8查询、FP4 KV张量、UE8M0组缩放和原生CDNA4 scaled-MFMA支持，通过优化的decode-attention内核实现高效推理。
   - 结果：在长上下文多轮agent工作负载上，UltraQuant在缓存压力的后期轮次将P50首次token延迟降低3.47倍（所有轮次平均2.3倍），并将输出吞吐量提升1.63倍（相对于FP8 KV基线）。

2. [Execution-State Capsules: Graph-Bound Execution-State Checkpoint and Restore for Low-Latency, Small-Batch, On-Device Physical-AI Serving](https://arxiv.org/abs/2606.20537v1)
   - 分类：Execution-State Checkpoint & Restore / Low-Latency Serving
   - 研究价值：值得精读
   - 关联：提出图绑定的执行状态胶囊，实现亚毫秒级完整状态恢复，在低延迟设备端服务中显著加速TTFT。
   - 痛点：主流LLM serving系统仅通过KV缓存重用前缀，但管理的是执行状态的一个位置片段（KV缓存）。在低延迟、小批次、设备端物理AI服务场景中（交互式LLM智能体、语音系统、机器人策略），需要完整的可恢复状态，包括KV、循环状态、卷积状态等，现有方法无法实现全局状态复用。
   - 方法：提出执行状态胶囊（execution-state capsules），一种图绑定的检查点与恢复机制，用于在提交边界完全恢复完整状态。FlashRT是白盒后端内核运行时，在NVIDIA CUDA上对捕获的图计划在连续静态缓冲区上执行，无块表间接寻址。胶囊可以快照、恢复、分支或回滚整个执行边界，包括KV、循环状态、MTP状态和元数据。
   - 结果：在RTX 5090上，胶囊恢复在存储状态级别字节精确，贪心解码下token一致。GPU驻留快照和恢复均为亚毫秒级。TTFT加速比从2k token的3.9倍提升到16k token的27倍。在Jetson AGX Thor和DGX Spark上也验证了相同性质。

3. [SSD: Spatially Speculative Decoding Accelerates Autoregressive Image Generation](https://arxiv.org/abs/2606.20543v1)
   - 分类：Speculative Decoding / Autoregressive Image Generation
   - 研究价值：值得精读
   - 关联：利用二维空间局部性提出空间推测解码，将自回归图像生成加速高达13.3倍。
   - 痛点：自回归模型将图像视作1D token序列，忽略了视觉信号固有的2D空间局部性，造成推理时的严重计算瓶颈。现有方法在1D序列中仅预测下一个token，未能利用空间相关性。
   - 方法：提出空间推测解码（Spatially Speculative Decoding, SSD），让模型同时预测相邻水平token和正下方token，利用二维空间相关性。通过将预测目标与图像自然几何对齐，SSD在保持高保真度的同时突破视觉推理中的内存墙。
   - 结果：在DPG-Bench和GenEval上，SSD将自回归图像生成加速高达13.3倍，同时保持高保真度。实验表明尊重图像的底层几何可以解锁大量计算效率。

**主要痛点**
- 现有KV缓存压缩方法在上下文密集agent场景中面临质量与效率权衡，且缺乏针对多轮任务的联合优化。
- 低延迟设备端服务中缺乏完整执行状态的复用机制，仅KV缓存片段复用不足。
- 自回归图像生成中忽略2D空间局部性造成推理严重计算瓶颈。
- 世界动作模型在非马尔可夫环境中性能下降，长历史导致时空成本线性增长。
- 长视频推理中密集处理整个视频成本高昂，稀疏字幕推理丢失时序信息。

**方法趋势**
- UltraQuant: 非对称K/V处理、Walsh-Hadamard旋转、FP4近似路径的4-bit KV缓存量化。
- Execution-State Capsules: 图绑定检查点与恢复，FlashRT在连续缓冲区执行无块表间接寻址。
- SSD: 利用2D空间相关性的推测解码，同时预测水平和垂直相邻token。
- MemoryWAM: 混合记忆设计（近期帧、事件锚点帧、gist token）和定制注意力机制。
- TimeProVe: 先轻量模块生成假设再VLM验证的Propose-and-Verify模式。

**结果信号**
- UltraQuant: 后期轮次P50 TTFT降低3.47倍，吞吐量提升1.63倍。
- Execution-State Capsules: 亚毫秒级恢复，TTFT加速比3.9x-27x。
- SSD: 自回归图像生成加速高达13.3倍，保持高保真度。
- MemoryWAM: 在长时程操控任务中优于VLA和WAM基线，计算效率更优。
- TimeProVe: VLM调用减少75%，推理成本降低93%，性能提升7.3%。
