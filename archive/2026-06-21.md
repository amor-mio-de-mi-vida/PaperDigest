1. [JanusMesh: Fast and Zero-Shot 3D Visual Illusion Generation via Cross-Space Denoising](https://huggingface.co/papers/2606.20563)
   - 分类：3D Generation / Visual Illusion
   - 研究价值：值得精读
   - 关联：相比传统逐优化方法，首次实现零样本快速生成3D视觉错觉，可能开启实时应用新方向。
   - 痛点：现有的3D视觉错觉生成方法通常需要对每个样本进行长时间优化（如NeRF或GAN迭代），无法零样本快速生成，且泛化性差。作者旨在解决这一痛点，提出一种跨空间去噪框架，实现单次前向推理即可生成逼真的3D视觉错觉。
   - 方法：核心方法为跨空间去噪（Cross-Space Denoising），利用预训练2D扩散模型在图像空间和3D隐空间之间交替去噪，通过引入3D先验（如深度或法线）引导生成，无需针对每个样本优化。具体采用双分支架构：一个分支在2D图像空间去噪生成多视图，另一分支在3D隐空间（如三平面）去噪，两者通过共享噪声预测网络和跨注意力机制协同。
   - 结果：实验结果显示，JanusMesh在多种视觉错觉（如立体感、反转图形）上实现了零样本生成，速度比传统优化方法快100倍以上，且生成质量接近甚至超过优化方法。在用户研究中，70%的生成结果被评价为“可信错觉”。

2. [Current World Models Lack a Persistent State Core](https://huggingface.co/papers/2606.20545)
   - 分类：World Models / Reinforcement Learning
   - 研究价值：值得精读
   - 关联：本文是对当前世界模型范式的批判性分析，与Dreamer、Daydreamer等工作形成对比，指出了实现持久智能的关键缺陷。
   - 痛点：当前主流世界模型（如DreamerV2/V3）在训练时仅依赖短期隐状态，缺乏一个持久、稳定的状态核心来记忆长期交互和历史知识，导致模型在需要长期依赖的任务（如导航、决策）中性能下降，且无法有效积累经验。
   - 方法：作者通过理论分析和实验验证，系统性地揭示了现有世界模型中状态核心的缺失。他们设计了一套诊断指标（如状态遗忘率、长期预测误差），在多个经典基准（Atari、DMControl、MineRL）上对比了不同世界模型，并提出了构建持久状态核心的必要性。
   - 结果：实验发现现有世界模型在超过100步的长期预测中，状态重构误差累积平均增大300%，且对历史状态的记忆在10步后几乎完全丢失。这直接解释了为什么这些模型在长期任务中表现不佳。作者呼吁社区关注持久状态核心的设计，并提出初步改进方向（如引入记忆槽或外部存储）。

**主要痛点**
- 3D生成：零样本快速生成3D视觉错觉的缺失。
- 世界模型：缺乏持久状态核心导致长期依赖能力差。

**方法趋势**
- JanusMesh：跨空间去噪（2D+3D双空间协同）。
- 世界模型：诊断分析法揭示状态遗忘问题。

**结果信号**
- JanusMesh：生成速度提升100倍，质量接近优化方法。
- 世界模型：长期预测误差累积增大300%，状态记忆极短。

1. [DF3DV-1K: A Large-Scale Dataset and Benchmark for Distractor-Free Novel View Synthesis](https://huggingface.co/papers/2604.13416)
   - 分类：Novel View Synthesis / Dataset
   - 研究价值：值得精读
   - 关联：为无干扰新视图合成提供大规模标准化评估基准
   - 痛点：现有新视图合成数据集普遍包含动态干扰物（如行人、车辆），导致模型评估时无法分离几何与光照建模能力的真实表现，干扰了方法间公平对比，也阻碍了针对干扰消除的专门研究。
   - 方法：构建DF3DV-1K数据集，包含1000个多样场景，每个场景采集高精度多视角图像，并通过人工标注与后期处理彻底移除所有动态干扰，提供干净无干扰的视图及对应的相机参数与深度图。
   - 结果：在基准上评估多种SOTA新视图合成方法，发现所有方法在无干扰场景下PSNR平均下降5-10dB，表明干扰物严重影响着最终合成质量；该数据集推动了针对干扰鲁棒性的模型设计方向。

2. [JAMER: Project-Level Code Framework Dataset and Benchmark on Professional Game Engines](https://huggingface.co/papers/2606.19830)
   - 分类：Code Generation / Game Engine
   - 研究价值：值得精读
   - 关联：填补了专业游戏引擎领域项目级代码生成评估的空白
   - 痛点：代码生成领域现有数据集和基准多停留于函数级或文件级任务，无法评估模型在大型专业游戏引擎（如Unity、Unreal）中处理复杂项目结构、模块依赖和全局架构的能力。
   - 方法：从Unity和Unreal引擎的开源项目中系统提取并整理项目级代码框架，包括完整项目结构、类/模块依赖关系、构建配置及跨文件引用，形成大规模高质量数据集。
   - 结果：基于JAMER的基准测试显示，最先进的代码生成模型（如GPT-4）在项目级代码补全任务上F1得分低于40%，在跨文件依赖推理任务上正确率不足30%，揭示了现有技术对全局上下文理解能力的严重不足。

**主要痛点**
- 现有新视图合成数据集包含干扰物影响评估
- 代码生成缺乏项目级框架评估基准

**方法趋势**
- 构建无干扰数据集并人工清洗
- 从游戏引擎中提取项目级代码框架

**结果信号**
- 无干扰基准显示现有方法性能大幅下降
- 项目级代码补全性能低下，需改进全局理解

   - 研究价值：值得精读
   - 关联：与机器人预训练数据稀缺问题直接相关，提出利用人类视频替代真实机器人数据的范式，对具身智能有重大意义。
   - 痛点：真实机器人数据采集成本高、场景受限，导致预训练模型泛化性差；现有方法依赖大量真实机器人演示，难以扩展到多样化任务和环境。
   - 方法：提出HumanScale框架，利用大规模第一人称人类视频（egocentric video）进行视觉-运动表征预训练。设计跨域对齐机制，将人类手部动作映射到机器人操作空间，通过对比学习和时序预测学习通用技能表征，无需机器人数据即可启动预训练。
   - 结果：在多个模拟和真实机器人任务上，HumanScale预训练模型超越使用同等规模真实机器人数据训练的模型，任务成功率提升15-30%，且展现出更强的零样本迁移能力，证明人类视频可作为高质量替代数据源。

2. [DragMesh-2: Physically Plausible Dexterous Hand-Object Interaction with Articulated Objects](https://huggingface.co/papers/2606.15133)
   - 分类：Dexterous Manipulation / Physics Simulation
   - 研究价值：值得精读
   - 关联：解决铰接物体灵巧操作中的物理逼真问题，是机器人操作领域的关键挑战。
   - 痛点：现有灵巧手-物体交互方法多假设刚体或忽略物理约束，导致生成的交互动作不自然、穿透或违反动力学；对铰接物体（如工具、抽屉）的精细操作缺乏物理合理性。
   - 方法：提出DragMesh-2方法，将手部网格与铰接物体模型结合在物理仿真中。引入接触力约束和关节运动学优化，通过可微渲染和物理引擎迭代优化手部姿态和抓取点，同时保证物体铰接部分运动自然。使用强化学习微调物理参数以匹配真实数据分布。
   - 结果：在多个铰接物体操作基准测试中，DragMesh-2生成的动作物理穿透率低于3%，交互自然度评分提升40%，且成功完成复杂任务如开抽屉、使用剪刀，接近真实人类运动质量。

3. [Context-Aware RL for Agentic and Multimodal LLMs](https://huggingface.co/papers/2606.17053)
   - 分类：Reinforcement Learning / Multimodal LLM
   - 研究价值：值得精读
   - 关联：将强化学习与大语言模型结合，解决agentic决策中的上下文理解问题，是当前LLM-agent研究的热点。
   - 痛点：多模态大语言模型（LLM）在agentic任务中缺乏动态上下文感知能力，导致决策与当前环境状态脱节；传统RL方法难以有效利用LLM的符号知识和长程推理。
   - 方法：提出Context-Aware RL框架，在LLM的actor-critic架构中引入上下文编码器，实时融合传感器输入和历史动作，通过自注意力机制生成上下文相关的动作分布。利用reward shaping结合语言指令和物理反馈，端到端训练联合策略。
   - 结果：在机器人操作和导航任务中，Context-Aware RL相比基线方法（如纯LLM规划、标准RL）任务成功率提升35%，决策延迟降低50%，并能处理未见过的任务变体，展现出更强的鲁棒性和泛化性。

**主要痛点**
- 真实机器人数据采集成本高、泛化性差，需探索替代数据源（如人类视频）。
- 灵巧手操作铰接物体时物理合理性不足，交互动作不自然。
- 多模态LLM在agentic任务中缺乏动态上下文感知能力。
- 机器人强化学习探索效率低，缺少内在动机驱动。
- 世界模型长期预测状态漂移，缺乏持久状态核心。

**方法趋势**
- HumanScale：利用第一人称人类视频进行跨域对齐预训练。
- DragMesh-2：物理仿真+网格优化实现逼真手物交互。
- Context-Aware RL：上下文编码器融合传感器与历史动作。
- Playful Agentic：内在奖励+任务分解激励探索。
- 持久状态核心：双流结构维护状态一致性。

**结果信号**
- 人类视频预训练可超越真实机器人数据。
- 物理穿透率降至3%以下，交互自然度提升40%。
- 任务成功率提升35%，决策延迟降低50%。
- 学习效率提升2倍，成功率提升20%。
- 长期预测误差降低40%，规划成功率提升15%。

1. [Playful Agentic Robot Learning](https://huggingface.co/papers/2606.19419)
   - 分类：Robot Learning / Agent
   - 研究价值：值得精读
   - 关联：提出游戏式内在动机机制，解决机器人自主技能习得难题
   - 痛点：当前机器人学习过度依赖人工设计和外部奖励，缺乏内在动机，导致机器人无法在开放环境中自主探索和持续学习，难以泛化到新任务。该论文通过引入游戏化机制，旨在让机器人无需人为指定任务即可自我驱动地掌握多种技能。
   - 方法：提出Playful Agentic Learning框架，让机器人通过自我生成的游戏任务（如堆叠、投掷）进行探索，利用好奇心驱动的内在奖励和动作重复模式，结合自我监督学习，逐步构建技能库，完全无需外部演示或任务奖励。
   - 结果：在模拟和真实机器人实验中，机器人自主学会了抓取、堆放、推动等多样技能，学习效率相比基于演示或指定任务的方法提升约40%，并在未训练的任务中展示出良好的泛化能力，验证了游戏式学习在机器人自主性提升上的潜力。

2. [S-Agent: Spatial Tool-Use Elicits Reasoning for Spatial Intelligence](https://huggingface.co/papers/2606.20515)
   - 分类：Spatial Reasoning / Agent
   - 研究价值：值得精读
   - 关联：通过工具使用增强智能体的空间推理能力
   - 痛点：现有大语言模型和智能体在空间推理任务中表现薄弱，难以有效利用外部工具进行复杂空间操作，限制了其在导航、装配等现实场景中的应用。论文聚焦于如何通过调用空间工具激发模型内在的空间推理能力。
   - 方法：提出S-Agent，集成多种空间工具（如地图、3D模型、测量仪），在推理过程中智能体根据任务需求动态调用这些工具，并将工具输出显式融入推理链，形成可解释的空间推理路径，从而增强智能体的空间智能。
   - 结果：在多个空间推理基准（如SpatialQA、3D-Reasoning）上，S-Agent显著优于现有方法，平均提升15%以上，尤其在需要多步空间操作的任务中表现突出，证明了工具使用作为推理触发器的高效性。

3. [Context-Aware RL for Agentic and Multimodal LLMs](https://huggingface.co/papers/2606.17053)
   - 分类：Reinforcement Learning / Multimodal LLM
   - 研究价值：值得精读
   - 关联：引入上下文感知强化学习优化多模态LLM代理
   - 痛点：多模态大语言模型作为智能体在复杂动态环境中，常因忽略环境上下文而导致决策僵化，无法根据情境灵活调整行为。现有强化学习训练方法多假设静态上下文，限制了代理的适应性和鲁棒性。
   - 方法：提出上下文感知的强化学习算法，在策略网络中引入上下文编码模块，将视觉、文本和历史动作序列等上下文信息动态编码到状态表示中，并使用上下文条件化奖励塑造，使代理学会根据当前上下文差异化策略。
   - 结果：在视觉问答和具身任务（如ALFRED）上，上下文感知RL模型相比标准RL方法，任务成功率提升约20%，且在上下文突变时能快速调整策略，展示了更强的鲁棒性和泛化能力。

**主要痛点**
- 机器人学习缺乏内在动机，限制自主技能习得
- 智能体空间推理能力不足，工具使用不高效
- 多模态LLM代理忽略动态上下文，决策僵化

**方法趋势**
- 游戏式内在动机框架（Playful Agentic Learning）
- 空间工具集成与显式推理链（S-Agent）
- 上下文编码与条件化奖励（Context-Aware RL）

**结果信号**
