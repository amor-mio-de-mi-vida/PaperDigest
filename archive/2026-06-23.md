1. [Vera: A Layered Diffusion Model for Content-Preserving Video Editing](https://arxiv.org/abs/2606.23610v1)
   - 分类：Video Editing / Layered Diffusion
   - 研究价值：值得精读
   - 关联：视频编辑
   - 痛点：现有视频编辑模型重置每个像素，常改变无需改动的角色或背景，缺乏内容保留机制。Vera通过分层扩散框架将编辑与内容保留分离，生成编辑层和alpha通道，确保背景和角色不变。
   - 方法：提出Vera框架，采用Mixture-of-Transformers（MoT）架构，包含两个DiT分别处理源视频和编辑层，通过联合自注意力交互。训练使用专门构建的高质量分层数据集，包含精确alpha通道和多样场景与动态。
   - 结果：在定量基准和人工偏好研究中，Vera在内容保留上优于领先开源模型，编辑质量具有竞争力。仅用48.6万帧分层训练数据即实现优异性能，证明了分层方法在视频编辑中的有效性。

2. [Scheduling Thoughts: Learning the Order of Thought in Diffusion Language Models](https://arxiv.org/abs/2606.23567v1)
   - 分类：Diffusion Language Model / Order Learning
   - 研究价值：值得精读
   - 关联：语言模型生成顺序
   - 痛点：掩码扩散语言模型的解码顺序通常靠启发式方法选取，严重影响生成质量但缺乏系统性优化。缺乏理论指导导致次优性能，无法自适应任务难度。
   - 方法：推导解码顺序策略优化的可处理上界，将顺序选择建模为策略优化问题。提出Self-Aware Scheduling（SAS），使用GRPO学习轻量顺序策略，适用任意顺序和半自回归解码，并支持灵活步数调度。
   - 结果：在Sudoku任务上准确率从82.0%提升至91.8%，GSM8K上pass@1从64%升至76%，MBPP从39.5%升至41%。一致超越启发式调度，表明学习解码顺序能显著提升扩散语言模型性能。

3. [DiT-Reward: Generative Representations for Text-to-Image Reward Modeling](https://arxiv.org/abs/2606.23626v1)
   - 分类：Reward Model / Text-to-Image
   - 研究价值：值得精读
   - 关联：奖励模型
   - 痛点：生成图像评估缺乏高质量奖励模型，现有HPSv3等方法依赖特定训练且成本高。预训练生成模型的表示能否直接用于奖励建模是开放问题，亟需高效可迁移方案。
   - 方法：将预训练文本到图像DiT转换为奖励模型，通过处理近干净图像潜在并聚合跨层表示。引入轻量学习头，在相同训练数据下与HPSv3对比，并探索骨干冻结和缩放行为。
   - 结果：DiT-Reward在四个偏好基准上超越HPSv3（HPDv2 85.6%，HPDv3 77.6%），冻结骨干时仍可提取偏好。用于优化SD3.5 Large时获得更佳真实感，推理速度提升1.65倍，证明生成DiT表示可迁移用于奖励建模。

**主要痛点**
- 视频编辑中内容保留与编辑分离困难
- 扩散语言模型解码顺序依赖启发式
- 生成图像评估缺乏高效可迁移奖励模型
- 动态重建易受遮挡和大变形
- 参考图像生成计算成本随参考数严重增加

**方法趋势**
- 分层扩散与MoT架构
- 策略优化学习解码顺序
- 利用生成DiT表示进行奖励建模
- 因果潜在条件与可变形3DGS
- 随机丢弃训练与任务感知选择

**结果信号**

- DiT-Reward在HPDv2达85.6%

### 高效训练与推理

1. [Tapered Language Models](https://arxiv.org/abs/2606.23670v1)
   - 分类：Language Model Architecture / Efficient Design
   - 研究价值：值得精读
   - 关联：提出简单有效的架构改进，可免费提升性能
   - 痛点：现代语言模型所有层参数均匀分配，但研究表明不同层贡献非均匀，后期层主要微调残差流而非转换。这种均匀分配浪费参数，限制了模型性能。需要一种在固定预算下按深度分配容量的方法。
   - 方法：提出Tapered Language Models (TLMs)，在固定总参数量下，将MLP宽度按深度单调递减（如余弦调度）。实验覆盖Transformer、Gated Attention、Hope-attention、Titans四种架构，在多个规模上验证。
   - 结果：在三种模型规模和四种架构下，TLM一致优于均匀宽度基线，在无额外参数或计算成本下改进了困惑度和下游基准性能。表明深度感知容量分配是简单有效的设计原则。

2. [Randomized YaRN Improves Length Generalization for Long-Context Reasoning](https://arxiv.org/abs/2606.23687v1)
   - 分类：Long-Context Training / Positional Encoding
   - 研究价值：值得精读
   - 关联：解决长上下文训练后泛化到更长序列的问题
   - 痛点：LLM通常在短序列上预训练，然后通过额外训练扩展到更长上下文，但即便如此，在面对远超出训练长度的序列时仍难以泛化。现有位置编码方法在分布外长度上性能下降严重。
   - 方法：提出Randomized YaRN，结合YaRN位置插值与随机位置编码和长度课程训练。在短上下文数据训练时，为token分配从更大位置范围采样的YaRN位置编码，使模型在短输入下也接触OOD位置表示。
   - 结果：在BABILong和MRCR基准上，使用<8K上下文训练，Randomized YaRN在16K到128K长度上持续提升推理性能，优于标准微调，在远OOD长度上增益最大。表明逐步暴露OOD位置分布是有效配方。

3. [SVD-Surgeon: Optimal Singular-Value Surgery for Large Language Model Compression](https://arxiv.org/abs/2606.23568v1)
   - 分类：Model Compression / SVD
   - 研究价值：值得精读
   - 关联：无需训练的模型压缩方法，优化SVD截断
   - 痛点：低秩压缩（SVD）简单有效，但现有方法仅关注如何分解和保留哪些成分，忽略了被截断的奇异值对模型损失的二阶影响。这导致压缩后模型性能下降。
   - 方法：引入SVD-Surgeon，将Optimal Brain Surgeon (OBS)框架应用于奇异值基础。将每个奇异值视为参数，对因截断而移除的奇异值，计算保留奇异值的闭式更新以补偿二阶损失。同时提供显著性指标用于选择剪枝哪些奇异值。
   - 结果：可直接叠加到现有SVD压缩方法上。应用于SVD-LLM，在OPT家族和LLaMA 2-7B上改进了困惑度-压缩权衡，无需任何重训练。

**主要痛点**
- LLM训练和推理中参数利用率低、长上下文泛化困难、模型压缩性能损失、KV缓存重复计算。
- 机器人操作中数据收集成本高、连续性操作难以训练、指令跟随退化。
- 多智能体系统提示优化效果不确定。

**方法趋势**
- 非均匀层容量分配（Tapered LMs）、随机位置编码（Randomized YaRN）、二阶最优SVD压缩（SVD-Surgeon）。
- 位置不变KV缓存（Kamera）、推理时并行聚合（SPIRAL）、锐度感知微调（Flatness）、判别器过程奖励（Success Visitation Matching）。
- 扩散模型解码顺序优化（Scheduling Thoughts）、多任务步调度（dVLA-RL）。

**结果信号**
- Tapered LMs在多架构上免费提升性能；Randomized YaRN显著提升长上下文推理；SVD-Surgeon压缩模型无重训练退化。
- Kamera大幅减少多模态KV重编码；SPIRAL推理扩展效率提升11倍；Flatness提升指令跟随60%。
- Scheduling Thoughts在Sudoku和数学推理上超越启发式；dVLA-RL在LIBERO达99.7%成功率。

1. [CoorDex: Coordinating Body and Hand Priors for Continuous Dexterous Humanoid Loco-Manipulation](https://arxiv.org/abs/2606.23680v1)
   - 分类：Robot Manipulation / Reinforcement Learning
   - 研究价值：值得精读
   - 关联：直接解决具身智能中全身协调灵巧操作的难题，方法具有通用性和可扩展性。
   - 痛点：现有类人机器人操作通常采用分步进行的'走走停停'模式，且使用低自由度夹爪，无法在移动中执行灵巧操作。高维度灵巧手（20-DoF）的控制与全身运动协调困难，缺乏有效的学习框架来同时处理身体和手的协同问题。传统强化学习方法在如此高维的接触丰富任务中难以训练，导致连续定位操作能力缺失。
   - 方法：CoorDex提出一种学习流水线，首先从模拟演示训练特权运动跟踪教师和手部专家，然后蒸馏为感知条件的潜在先验。这些先验作为残差强化学习的动作空间，通过共享任务上下文和分离身体-手部残差头的协调潜在残差策略组合先验，保留自然全身运动的同时改善手指级接触可靠性。该设计将高维控制转化为低维协调残差，使得高维接触丰富的定位操作变得可训练。
   - 结果：在Unitree G1人形机器人上实现非停止瓶子抓取携带、移动中打开冰箱门和方块取放等任务，这是首次在行进中完成灵巧操作。消融实验表明，联合空间PPO、手部直接控制和单一潜在预测等方法均失败，而潜在先验和协调残差结构是成功的关键，验证了方法的有效性。

2. [dVLA-RL: Reinforcement Learning over Denoising Trajectories for Discrete Diffusion Vision-Language-Action Models](https://arxiv.org/abs/2606.23623v1)
   - 分类：Vision-Language-Action / Reinforcement Learning
   - 研究价值：值得精读
   - 关联：首次将强化学习应用于离散扩散VLA模型，拓展了VLA策略优化范式。
   - 痛点：离散扩散VLA（dVLAs）模型作为一种新颖的机器人操控框架，目前仅通过监督微调（SFT）训练，未探索强化学习（RL）进一步优化策略的潜力。传统RL方法难以应用于dVLAs，因为最终动作的边际概率不可处理，且去噪步数可变带来训练复杂度。如何在保持dVLAs统一离散表示优势的同时，利用RL提升策略性能是一个关键挑战。
   - 方法：dVLA-RL将去噪过程建模为马尔可夫决策过程（MDP），以生成路径的联合概率作为学习目标，代替边际概率。通过数学推导，将路径概率表示为逐步转移的乘积，从而自然地适应变量去噪步数。此外，引入统一步数调度方法，针对不同任务复杂度分配去噪步数，在保证成功率的同时提高计算效率。整个框架通过策略梯度优化，端到端训练。
   - 结果：在LIBERO基准上达到99.7%的成功率，在RoboTwin 2.0上比SFT基线提升30.6%，与基于世界动作模型的强基线竞争。实验表明，RL训练显著改善了策略的稳定性和泛化能力，且统一步数调度有效平衡了成功率和计算成本。

3. [Flatness Preserves Instruction Following in Vision-Language-Action Models](https://arxiv.org/abs/2606.23641v1)
   - 分类：Vision-Language-Action / Optimization
   - 研究价值：值得精读
   - 关联：揭示了VLA模型微调中的指令失明问题并提出简单有效的解决方案。
   - 痛点：视觉-语言-动作（VLA）模型在有限机器人数据上微调时，损失景观尖锐，导致模型对权重扰动敏感，从而忽视语言指令而依赖视觉捷径，出现'指令失明'现象。现有方法要么需要额外数据，要么修改架构，而缺乏一种轻量级、与数据无关的解决方案来保持指令跟随能力。
   - 方法：提出在VLA微调中应用锐度感知最小化（Sharpness-Aware Minimization, SAM），通过同时最小化损失值和损失锐度，学习一个更平坦的损失景观。该方法不修改模型架构，不增加数据，仅在优化过程中加入扰动和梯度对齐步骤，使得微调后的模型对权重空间的扰动更鲁棒，从而保留预训练的语言-视觉对齐能力。
   - 结果：在多个仿真和真实世界基准上，指令跟随性能提升超过60%，且与现有引导技术（如行为克隆、数据增强）互补。实验证明，平坦最小值对应的模型在指令变化和视觉干扰下更稳定，显著减轻了指令失明问题。

**主要痛点**
- 具身智能中数据收集效率低、成本高
- VLA模型指令跟随能力不足，存在指令失明
- 全身协调灵巧操作缺乏有效学习框架
- 稀疏奖励导致强化学习信用分配困难

**方法趋势**
- 自动化数据收集系统（AutoDex）
- 锐度感知最小化保持指令跟随（Flatness）
- 潜在先验+残差强化学习协调身体手部（CoorDex）
- 将扩散去噪过程建模为MDP进行RL训练（dVLA-RL）
- 成功访问匹配转换为密集奖励（Process Reward）

**结果信号**
- 吞吐量提升4.8倍，抓取成功率76%（AutoDex）
- 指令跟随性能提升60%以上（Flatness）
- 实现非停止灵巧操作，消融验证关键组件（CoorDex）
- LIBERO 99.7%成功率，RoboTwin提升30.6%（dVLA-RL）
- 显著加速RL微调（Process Reward）

1. [Self-Compacting Language Model Agents](https://arxiv.org/abs/2606.23525v1)
   - 分类：Language Model Agents / Context Management
   - 研究价值：值得精读
   - 关联：与Agent系统中的上下文管理紧密相关，解决长轨迹效率问题。
   - 痛点：长期运行的LLM智能体轨迹会积累陈旧内容，导致上下文窗口溢出并锚定后续生成，而现有固定间隔压缩方法可能在中间推理或搜索过程中丢弃部分结果。模型自身缺乏判断何时需要压缩的能力，导致无效或有害的压缩行为。该问题严重制约了智能体在复杂、多步任务中的部署，因为每次压缩都可能损失有价值的状态信息。
   - 方法：提出SelfCompact框架，通过两个协同元素实现自适应压缩：一是压缩工具，由模型自主调用以总结积累的上下文；二是轻量级规则，指定触发压缩的时机（子任务完成或轨迹收敛）和抑制时机（中间推导或陷入困境）。两者结合使模型无需微调或外部监督即可有效压缩。框架在多个开源模型上验证，无需训练即可迁移。
   - 结果：在六个基准（数学推理和智能体搜索）和七个模型上，SelfCompact匹配或超越固定间隔摘要方法，同时token成本降低30-70%。在数学任务上提升最高18.1个百分点，在智能体搜索上提升5-9个百分点。实验还揭示了元认知差距：未受提示的模型无法可靠判断上下文是否退化，而轻量级规则可以弥补这一差距。

2. [AIR: Adaptive Interleaved Reasoning with Code in MLLMs](https://arxiv.org/abs/2606.23678v1)
   - 分类：Multimodal LLM / Reinforcement Learning / Code Reasoning
   - 研究价值：值得精读
   - 关联：增强MLLM在数值计算任务中的推理能力，适用于Agent需要调用代码的场景。
   - 痛点：现有MLLM在视觉感知任务中主要依赖预定义启发式进行工具使用，缺乏处理数值计算任务的能力，且无法自适应地在推理过程中交替使用视觉理解和代码执行。这限制了MLLM在需要精确计算的领域（如科学计算、数据分析）中的应用。此外，强化学习训练数据构建和奖励设计面临挑战。
   - 方法：提出三组件解决方案：1) 两阶段冷启动数据构建管线，生成代码增强的复杂数值计算任务数据；2) 数据过滤策略用于强化学习数据集筛选；3) 自适应工具调用策略，利用组约束奖励函数引导模型在推理轨迹中交织代码执行。通过扩展强化学习训练，使模型学会何时调用代码以及如何整合结果。
   - 结果：强化学习训练后，评估基准平均提升6.1个百分点，其中交织推理样本的准确率提升9.9个百分点，工具调用成功率超过95%。这表明自适应交织推理显著增强了MLLM的数值计算能力。

3. [Semantic Browsing: Controllable Diversity for Image Generation](https://arxiv.org/abs/2606.23679v1)
   - 分类：Image Generation / Controllable Diversity / VLM Agent
   - 研究价值：值得精读
   - 关联：为图像生成Agent提供可控多样性方法，可应用于创意设计场景。
   - 痛点：现代文本到图像模型严格遵循提示，导致生成样本缺乏多样性，且现有提高多样性的方法产生的是随机或无意义的变化，而非用户可理解的结构化设计选择。用户无法系统地探索图像空间中的有意义变化，降低了创意工作的效率。
   - 方法：利用文本到图像模型训练时使用的详细标题，将多样性控制从噪声层面转移到文本层面。使用视觉语言模型（VLM）理解完整场景上下文，并通过agentic workflow强制产生结构化的语义变化，确保每种变化对应特定的、用户可理解的语义决策（如改变颜色、纹理、物体位置等）。用户可以在导航式图像画廊中浏览。
   - 结果：方法生成多样且可导航的设计空间，每个变化都可解释，用户能够系统地探索有意义的变体，实现了真正的创意探索体验。

**主要痛点**
- 长Agent轨迹中上下文窗口有限，固定压缩策略可能丢失中间结果
- 多模态大模型在数值计算任务中缺乏自适应代码推理能力
- 图像生成模型缺乏可控、可解释的多样性生成方法
- 多智能体系统提示优化面临搜索空间爆炸和效果不确定性
- 具身智能体缺乏统一框架，难以在真实机器人上部署

**方法趋势**

- 关键词：alignment, pretest question, pedagogical quality, ai-assisted workflow for pretest, findings show that human-machine, rubric-based evaluation and iterative
- 链接：<https://arxiv.org/abs/2606.23629v1>

**原始摘要**

Designing effective pretest questions is challenging at scale: high-quality questions require careful calibration of openness, cognitive depth, and alignment with learning objectives, yet generating and evaluating them manually is time-consuming. We present an AI-assisted workflow for pretest question development that combines automated generation, rubric-based evaluation, and iterative selection. Because the workflow relies on machine evaluation to filter questions at scale, we investigate the alignment between human and machine judgments across a 2x2 design varying rubric operationalization and evaluation mode. Our findings show that human-machine disagreements are systematic rather than random, that rubric revision has a larger effect on alignment than rationale-first evaluation, and that the two interventions are complementary. These findings highlight that scalable AI-assisted pretesting depends not only on generation capability but on how pedagogical quality is operationalized for machine interpretation.

### 6. [Decentralized Autonomous Traffic Management through Corridor Networks](https://arxiv.org/abs/2606.23585v1)

- 来源：arXiv
- 关键词：reinforcement learning, traffic density, autonomous traffic, corridor networks., traffic management, autonomous aircraft
- 链接：<https://arxiv.org/abs/2606.23585v1>

**原始摘要**

As autonomous aircraft are introduced at scale and traffic density increases, centralized management becomes insufficient to coordinate the large numbers of crewed and uncrewed aircraft. Dedicated Advanced Air Mobility (AAM) corridors have therefore been proposed for organizing high-density autonomous traffic flows. The desire to scalably provide autonomous aircraft flexibility in trajectory planning motivates the development of decentralized approaches to traffic management in AAM corridors. In this work, we extend a multi-agent reinforcement learning (MARL) approach to address the challenge of decentralized traffic flow management in air corridor networks. We test policies trained in a single-corridor setting on increasingly complex multi-corridor networks with combinations of merges and splits in a zero-shot manner. Experimental results demonstrate that learned behaviors transfer well to scenarios with varying traffic density, network geometry, and heterogeneous vehicle performance, without needing centralized coordination or model retraining. We evaluate system-level performance in terms of conformance to corridor boundaries, completion rates, average speeds, distance traveled, and maintenance of inter-aircraft separation. We find that although our policies require only locally coordinated entry, traversal, and exit behaviors, they collectively produce desirable traffic flows through the corridor network.

### 7. [HoloAgent-0: A Unified Embodied Agent Framework with 3D Spatial Memory](https://arxiv.org/abs/2606.23565v1)

- 来源：arXiv
- 关键词：holoagent-0, unified embodied agent framework, LLM agents, spatial memory, embodied agentos, agent framework for real-world
- 链接：<https://arxiv.org/abs/2606.23565v1>

**原始摘要**

LLM agents follow a practical execution loop in digital environments: they reason over structured states, invoke tools, inspect feedback, and revise actions. Extending this loop to physical robots is difficult because physical execution is continuous, embodiment-dependent, uncertain, and constrained by safety. Existing embodied-AI systems have advanced manipulation, spatial understanding, navigation, and humanoid control, but these capabilities often remain specialized modules or loosely coupled decision loops. In this work, we introduce HoloAgent-0, a unified embodied agent framework for real-world robot deployment. Embodied AgentOS converts language instructions into executable skill graphs, schedules robot resources, monitors execution, and triggers clarification or re-planning from runtime feedback. HoloAgent-0 organizes heterogeneous robot models and controllers through three coupled layers: Embodied AgentOS for closed-loop execution, 3D spatial memory for physical world grounding, and embodied skills for robot action. We deploy HoloAgent-0 on real hardware and evaluate its spatial memory, long-horizon navigation, and closed-loop execution across motion generation, object search, cross-robot coordination, and mobile manipulation.

### 8. [Self-Compacting Language Model Agents](https://arxiv.org/abs/2606.23525v1)

- 来源：arXiv
- 关键词：agent, fine-tuning, TOOL the model invokes, allows the model itself, self-compacting language, lightweight rubric
- 链接：<https://arxiv.org/abs/2606.23525v1>

**原始摘要**

Long agent traces composed of chains of thought and tool calls accumulate stale content that anchor subsequent generations, and eventually outgrow the context window. Existing scaffolds mitigate it with fixed-interval compaction triggered at a token threshold. Such triggers pay no heed to trajectory structure, risking discard of partial results mid-derivation or mid-search. We propose SelfCompact, a scaffold that allows the model itself to decide when and how to compact. Specifically, it pairs two inference-time elements: (i) a compaction tool the model invokes to summarize the accumulated context, and (ii) a lightweight rubric specifying when to fire (a sub-task has resolved, or the trajectory is converging) and when to suppress (mid-derivation, or when stuck). Both are needed. The tool alone is unevenly used across open-weight models, often invoked at unhelpful moments or not at all; the rubric alone cannot act. Together, they elicit effective adaptive compaction without any fine-tuning or external supervision. We present empirical results on six benchmarks (competitive math and agentic search) and seven models. Our results show that SelfCompact matches or exceeds fixed-interval summarization at a fraction of the token cost, improving over a no-summarization baseline by up to 18.1 points on math and 5-9 points on agentic search at 30-70% lower per-question cost. Our results expose a meta-cognitive gap: although unprompted models cannot reliably tell when their own context is rotting, a lightweight rubric closes this gap, reframing when to compact as a capability that scaffolds can supply without training.

### 9. [Concordia: JIT-Compiled Persistent-Kernel Checkpointing for Fault-Tolerant LLM Inference](https://arxiv.org/abs/2606.23521v1)

- 来源：arXiv
- 关键词：fault-tolerant LLM inference., LLM agents, attention, LLM inference. long-running LLM, substrate for fault-tolerant LLM, state region concordia jit-compiles
- 链接：<https://arxiv.org/abs/2606.23521v1>

**原始摘要**

Long-running LLM agents keep valuable state resident on GPUs: KV caches, request schedulers, communication state, and sometimes online adapters. Losing this state after a GPU or communicator failure can discard minutes to hours of work, yet existing recovery mechanisms either restart the whole serving stack or require application-specific checkpoint logic inside every attention and runtime component. This paper argues that fault tolerance for such workloads needs a GPU-resident execution context: checkpoint hooks must run at device synchronization points, observe binary kernels that frameworks and libraries actually execute, and recover without putting the host CPU on the critical path. We present Concordia, a runtime that uses a device-resident persistent kernel as the substrate for fault-tolerant LLM inference. Concordia interposes on GPU module loading and supports PTX- and SASS-level instrumentation, allowing checkpoint and pause hooks to be inserted below framework code and library boundaries. For each registered LLM state region, Concordia JIT-compiles a specialized delta-checkpoint handler -- for example, a KV-block scanner, adapter-page scanner, or recovery applier -- and hot-swaps it into the persistent kernel's operator table. The persistent kernel consumes a lock-free ring buffer of compute, checkpoint, append-log, and recovery tasks, so the same always-on executor triggers dirty-page detection, stages deltas, and appends committed records to a CPU-visible log in CXL memory or host DRAM.
