1. [WorldDirector: Building Controllable World Simulators with Persistent Dynamic Memory](https://arxiv.org/abs/2607.02517v1)
   - 分类：Video Generation / World Model
   - 研究价值：值得精读
   - 关联：持久动态记忆世界模型，与Align4D互补，一个侧重可控，一个侧重多模态对齐。
   - 痛点：现有视频世界模型将物理动态与像素渲染纠缠，依赖连续视觉观察维持运动，难以实现持久动态物体记忆和自由视角探索。具体痛点：当物体长时间离开视野后重新出现时，模型无法保持其精确视觉身份，且缺乏对复杂长事件的显式控制。
   - 方法：提出WorldDirector框架，显式解耦语义运动编排和视觉生成：利用大语言模型（LLM）协调3D轨迹与相机运动，将编排后的轨迹作为控制信号输入视频生成模型；引入持久动态记忆机制，在物体离开视野时保存其身份信息，使其在重新进入场景时能恢复完整外观。
   - 结果：实验证明该方法支持合成复杂且长时的动态事件，在可控性和持久动态物体记忆上达到前所未有的效果。与现有世界模型对比，在多个视频生成指标（如FVD、一致性）上有显著提升，成功保持了动态物体在长时间遮挡后的视觉身份不变。

2. [Alignment Is All You Need For X-to-4D Generation](https://arxiv.org/abs/2607.02516v1)
   - 分类：4D Generation / Multimodal Alignment
   - 研究价值：值得精读
   - 关联：统一多模态到4D的生成框架，包含新数据集，与WorldDirector在视频生成领域互补。
   - 痛点：任意用户定义模态到4D（X-to-4D）生成面临数据集构建成本高和现有方法可扩展性有限的挑战。难以将文本、图像、视频等多种输入模态协调地转换为一致、动态的4D内容（视频+3D），且缺乏统一基准。
   - 方法：提出Align4D框架，将任意模态输入翻译为连贯的视频-3D对。关键技术包括：对象距离对齐（搜索与视频对齐和多视角对齐的对象距离，以协调4D渲染与视频及多视角扩散模型先验）、运动-几何联合对齐（通过同步的视频和3D输入约束已知和未知视图）、异步优化（解耦高斯属性与变形网络训练以增强运动与几何保真度）。同时提出X4D数据集用于基准测试。
   - 结果：在X4D和Consistent4D数据集上，Align4D在X-to-4D生成质量和一致性上达到SOTA，显著优于现有方法。实验表明其能有效将不同模态输入转换为高质量、时间一致的4D内容，并提供了可靠的全套基准。

3. [PointDiT: Pixel-Space Diffusion for Monocular Geometry Estimation](https://arxiv.org/abs/2607.02515v1)

   - 结果：超越复杂潜在扩散模型，几何更锐利，在模糊区域（如透明物体）更鲁棒。

**主要痛点**

- 多模态到4D生成缺乏统一框架和数据集

- 单步图像生成质量受限，分布匹配易被单编码器欺骗
- 单图像3D重建依赖复杂架构和潜在空间压缩

### 高效训练与推理

1. [OrbitQuant: Data-Agnostic Quantization for Image and Video Diffusion Transformers](https://arxiv.org/abs/2607.02461v1)
   - 分类：Diffusion Model Quantization / PTQ
   - 研究价值：值得精读
   - 关联：首次实现数据无关量化，适用于多种扩散模型，无需校准数据，推进低比特部署。
   - 痛点：扩散Transformer（DiT）在图像和视频生成中推理成本高昂，后训练量化（PTQ）是常用加速手段。然而，DiT的激活分布在不同时间步、提示和引导分支中显著变化，迫使现有方法为每个新检查点或模态重新拟合校准数据。这种依赖性导致部署效率低下，且低比特（如W2A4）量化质量难以保证。OrbitQuant旨在实现数据无关的量化，无需校准数据即可适用于任意DiT模型。
   - 方法：OrbitQuant提出在归一化旋转基中进行量化。通过随机置换块Hadamard（RPBH）旋转，将激活值变换到固定均值和方差的坐标系中，使得所有时间步、提示和层共享同一个Lloyd-Max码书。对于权重行，预先吸收旋转到权重矩阵中，运行时仅需对激活进行正向旋转，消除校准需求。该方法从图像模型零迁移到视频模型，无需调参。
   - 结果：在FLUX.1、Z-Image-Turbo、Wan 2.1和CogVideoX等最新DiT模型上，OrbitQuant在多个低比特设置下达到了PTQ的SOTA。首次将图像DiT的PTQ推进到W2A4并保持可用生成质量，推理速度显著提升。该方法无需每模态或每检查点校准，大幅降低部署成本。

2. [DemoPSD: Disagreement-Modulated Policy Self-Distillation](https://arxiv.org/abs/2607.02502v1)
   - 分类：Self-Distillation / LLM Reasoning
   - 研究价值：值得精读
   - 关联：解决自蒸馏中的特权信息泄露问题，提升跨域泛化，可应用于LLM推理训练。
   - 痛点：在线自蒸馏（OPSD）中，教师模型利用特权信息（如未来token）提供密集token级监督，导致学生模型编码答案相关捷径（特权信息泄露），抑制探索能力，并在跨域任务中泛化下降。现有方法难以同时解决泄露和探索问题。
   - 方法：DemoPSD提出选择性采纳教师指导，构建反向KL重心目标，即教师与学生分布的加权几何组合，权重由每个token位置的分布差异自适应决定。通过差异度量控制混合程度，实现训练时逐步偏向教师但保留学生分布。理论证明该方法能减弱特权信息泄露并保持探索能力。
   - 结果：在SciKnowEval四个科学领域上，DemoPSD优于GRPO和SDPO，在保持更高训练熵的同时，在分布外GPQA基准上展现出更强的鲁棒性。实验证实了泄露抑制和探索保留的有效性。

3. [WattGPU: Predicting Inference Power and Latency on Unseen GPUs and LLMs](https://arxiv.org/abs/2607.02391v1)
   - 分类：GPU Power Prediction / LLM Inference
   - 研究价值：值得精读
   - 关联：提供基于元数据的功耗和延迟预测工具，辅助LLM推理部署优化。
   - 痛点：随着LLM推理能耗快速增长，数据中心运营商需要选择合适的GPU-LLM组合以优化能效，但缺乏工具预测未经验证的组合的功耗和延迟。现有方法要求对每对组合进行profiling，成本极高且难以扩展到新硬件。运营商需要一种无需硬件访问、基于公开元数据的预测方法。
   - 方法：WattGPU开发了两个预测模型：平均GPU功耗和token间延迟(ITL)。仅使用公开的LLM元数据（参数量、层数等）和GPU规格（计算能力、内存带宽等），无需任何profiling或硬件访问。通过留一GPU和留一LLM交叉验证评估，模型在8种GPU和42种LLM上测试，涵盖0.1B-27B参数。
   - 结果：功耗预测中位绝对误差≤3.4%（离线）和≤13.5%（服务器），延迟预测≤8.5%（服务器）。与物理基线（TDP和roofline）相比，误差降低约2-4倍。GPU排名相关性Kendall τ≥0.76，支持运营商快速选择最优组合。

**主要痛点**
- 扩散模型推理成本高，量化需校准数据，部署碎片化
- 自蒸馏存在特权信息泄露，抑制探索和泛化
- LLM推理功耗未知，缺乏预测工具，组合选择困难
- 具身模型部署缺乏统一运行时，边缘设备效率低
- VLA模型训练依赖大量专家演示，数据稀缺

**方法趋势**
- 数据无关量化（OrbitQuant）
- 自适应混合蒸馏（DemoPSD）
- 基于元数据的功耗延迟预测（WattGPU）
- 便携推理运行时（Embodied.cpp）
- 任务无关预训练（Learning to Move）

**结果信号**
- 扩散模型量化达到W2A4可用，无需校准（OrbitQuant）
- 自蒸馏超越GRPO/SDPO，泛化更强（DemoPSD）
- 功耗预测误差<4%，延迟<9%（WattGPU）
- 具身模型推理成功率>90%，内存降低72%（Embodied.cpp）
- VLA预训练数据量减少10倍，性能匹配（Learning to Move）

1. [What LLM Agents Say When No One Is Watching: Social Structure and Latent Objective Emergence in Multi-Agent Debates](https://arxiv.org/abs/2607.02507v1)
   - 分类：Multi-Agent / Social Dynamics
   - 研究价值：值得精读
   - 关联：揭示了多智能体系统中社会结构对隐性目标的影响，对Agent对齐和安全具有重要启示。
   - 痛点：现有LLM智能体评估主要关注明确目标，忽视了社会结构（如角色、受众、关系）如何驱动智能体产生与私下意图不一致的公开表达，即潜在目标涌现。这种不一致可能导致系统行为偏离设计意图，带来安全风险。论文提出需要检测智能体在无显式指示下涌现的隐藏目标。
   - 方法：提出双通道辩论框架（dual-channel debate framework），智能体在同一条件下同时生成公开话语（进入共享历史）和私下（OTR）回应（记录但不展示给对方）。通过10个模型、3个场景、每个场景5种变体进行系统实验，使用立场分析、语义相似度、自然语言推理和问卷调查四种聚合分析测量公开-私下分歧。
   - 结果：在对齐诱导设置下，目标智能体的公开-私下决策分歧从约3%基线提升至约40%。四类分析一致显示显著分歧。部分OTR回应明确将公开顺应归因于关系压力（如职业风险或赞助义务）。结论表明：智能体评估应超越显式目标，检测涌现的隐藏目标。

2. [Reasoning effort, not tool access, buys first-try reliability in agentic code generation: an observational study](https://arxiv.org/abs/2607.02436v1)
   - 分类：Code Generation / Reasoning
   - 研究价值：值得精读
   - 关联：实证对比了推理努力与工具访问对代理代码生成首次成功的影响，为系统设计提供关键指导。
   - 痛点：代理编码助手常被赋予额外能力（如浏览器测试工具、设计提示），但缺乏严格实验证明这些能力是否真正提升软件质量和首次可靠性。实践者面临选择：增加工具还是提升推理能力？需要明确哪种投入更有效。
   - 方法：设计对照实验：90个独立代理构建同一应用（实时回溯板），依据14条功能标准（42分满分）和视觉质量评分。变量包括模型代数、代理框架、推理努力级别（High vs xHigh）、测试工具、设计提示。通过准则级分析识别失败模式。
   - 结果：xHigh推理努力使首次完美运行率从28%提升至89%，纠正提示减少约5倍，成本仅增加9-29%。测试工具虽增加42-68%成本却未提升功能分数或可靠性。设计提示提升视觉质量但未改善功能。主要失败模式是容器部署缺陷（首次失败率44%），且受模型代数影响大。结论：匹配修复与失败原因，薄弱推理是主因，应通过更强模型或更高推理努力解决。

3. [EvoPolicyGym: Evaluating Autonomous Policy Evolution in Interactive Environments](https://arxiv.org/abs/2607.02440v1)

   - 研究价值：值得精读
   - 关联：提出自主策略演化评估框架，填补了现有评估仅关注最终得分的空白，提供轨迹级诊断。
   - 痛点：自主智能体需通过反馈迭代改进可执行策略，但现有评估常简化为最终得分，或与开放软件工程进步混淆。缺乏受控评估设置来分离代理如何分配预算、转换反馈为参数调优等细粒度行为。
   - 方法：提出自主策略演化（Autonomous Policy Evolution）受控设置：一个 harness-model 代理在固定交互预算下重复编辑可执行策略系统。基于此构建EvoPolicyGym基准，包含16个紧凑交互RL环境，评估代理如何迭代改进策略。提供轨迹级诊断指标。
   - 结果：GPT-5.5在EvoPolicyGym套件中取得最强聚合排名和所有16个环境的前两名。轨迹分析揭示代理在预算分配、反馈转换和参数调整方面的策略差异。强自主策略演化不仅依赖于孤立任务胜利，还依赖于发现任务合适机制并在有限反馈下精炼策略。
