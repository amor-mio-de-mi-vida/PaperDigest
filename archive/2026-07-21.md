1. [The Many Senses of Visual Similarity: A Text-Prompted Image Perceptual Metric](https://arxiv.org/abs/2607.18237v1)
   - 分类：Image Perceptual Metrics / Vision-Language
   - 研究价值：值得精读
   - 关联：为生成式AI评估提供细粒度自动度量，可直接用于评价图像/视频生成模型的感知质量。
   - 痛点：现有视觉相似性度量将多维度相似性（如形状、颜色）压缩为单一标量，无法根据文本条件化特定语义方面，导致评估与人类主观判断不一致，且缺乏可解释性和灵活性。这在生成式AI领域尤其突出，因为生成结果往往需要在特定属性（如风格、内容）上与参考图像对齐，但现有度量无法捕捉这种细粒度差异。
   - 方法：收集大规模人类相似性判断三元组数据集，每个三元组标注多个自由形式语义方面（如形状、颜色、纹理）。基于前沿视觉语言模型（如CLIP）微调，提出文本提示图像感知相似性度量TPIPS，使模型根据文本提示输出条件化相似性分数，实现多语义维度可控度量。
   - 结果：在多个感知相似性基准上，TPIPS更贴近人类共识，超越现有度量。在文本引导检索、组合搜索和生成模型细粒度评估中展现出新能力，例如检索更符合文本描述的图像，以及评估生成模型在不同语义维度上的表现。代码、数据和模型已开源。

2. [HOMIE: Human-object Centric Video Personalization via Multimodal Intelligent Enchancement](https://arxiv.org/abs/2607.18217v1)
   - 分类：Video Generation / Subject-Driven Personalization
   - 研究价值：值得精读
   - 关联：直接解决生成式AI中人物-物体交互视频个性化生成的核心挑战，提升主体保真度和交互准确性。
   - 痛点：现有主体驱动视频生成方法在人物-物体交互个性化（HOCVP）中难以平衡主体保真度和交互准确性，尤其当物体为抽象概念（如logo）时，保真度低；且缺乏理解多模态参考（如OCR图、多视角输入）内隐对应关系的机制，导致帧间不一致和交互模式错误。
   - 方法：提出统一处理主体间和主体内输入设置的HOMIE框架。关键创新包括：全局多模态引导，在self-attention中对齐MLLM提取的语义特征与VAE令牌，增强交互模式理解；模态参考嵌入，区分MLLM特征与VAE令牌，并关联主体内参考图像令牌，提升主体保真度。无需昂贵重对齐即可利用MLLM知识。
   - 结果：在多种人物-物体中心视频个性化任务（如人物与特定物体交互、logo动画）上达到SOTA性能，主体保真度和交互准确性显著优于现有方法。大量实验验证了框架的有效性和泛化能力。项目页面已公开。

3. [Three-Body Scattering for Generative Modeling](https://arxiv.org/abs/2607.18198v1)
   - 分类：Generative Models / One-Step Generation / Score-based
   - 研究价值：值得精读
   - 关联：提出新的生成模型范式，实现高效一步生成，对扩散模型和GAN的发展具有理论指导意义。
   - 痛点：现有生成模型依赖对抗判别器（GAN）、预设噪声到数据路径（扩散模型）或自回归分解，缺乏基于分布能量的直接一步回归监督。一步生成方法（如GAN）训练不稳定，扩散模型需要多步采样，效率低。需要一种兼具理论优美性和实用高效性的生成框架。
   - 方法：提出三步散射建模（TBSM），将能量距离转化为每射弹的恒定大小交互：每个射弹（生成样本）被一个真实样本吸引，被一个独立生成样本排斥，其条件期望等于2-Wasserstein梯度流速度。通过在线跟踪条件期望降低场噪声，实现直接回归监督。训练中，每批使用冻结目标，产生O(B)个样本级损失。
   - 结果：在ImageNet-256上，使用像素空间PixelDiT-XL实现FID=2.23，使用潜在空间DiT-XL实现FID=1.63，均为单步生成（NFE=1）。建立了扩散监督、漂移动力和GAN目标的统一设计图，代码已开源。该方法在效率与质量间取得良好平衡，启发新研究方向。

**主要痛点**
- 现有视觉相似性度量缺乏条件化语义能力，评估不精准
- 人物-物体交互视频生成中主体保真度和交互准确性难以平衡
- 一步生成缺乏有效的分布能量监督
- 实时多模态应用部署优化困难
- 商用自动驾驶系统罕见故障发现困难

**方法趋势**
- 文本提示感知度量TPIPS，基于VLM微调
- 全局多模态引导和模态参考嵌入的HOMIE框架
- 三步散射建模，通过吸引-排斥交互实现一步生成监督
- 链式程序引导Agent进行多轮部署优化
- AST和DiFS故障发现结合PCA分析

**结果信号**
- TPIPS更贴近人类感知，在检索和生成评估中展现新能力
- HOMIE在多种人物-物体视频个性化任务中达到SOTA
- TBSM在ImageNet-256实现一步生成FID=2.23/1.63

1. [TRIM: Reducing AI-Generated CodeSlop via Agent Trajectory Minimization](https://arxiv.org/abs/2607.18161v1)
   - 分类：Software Engineering / Agent
   - 研究价值：值得精读
   - 关联：直接解决Agent代码生成的冗余问题，具有广泛适用性
   - 痛点：编码代理生成的代码倾向于冗余，这种冗余源于代理自身的搜索过程——为寻找解法而积累的推测性编辑、废弃假设和临时修改最终残留在最终补丁中。单个补丁影响不大，但随着代理接管代码库的越来越多部分，冗余累积速度超过清理速度，导致代码库难以维护。现有方法如Delta Debugging直接最小化补丁差异，但验证成本高。
   - 方法：提出TRIM算法，通过最小化代理轨迹间接减少CodeSlop。核心思想是不直接优化最终代码的冗余，而是优化代理的搜索轨迹长度，从而间接减少不必要编辑的残留。TRIM在保持性能的前提下，大幅降低了冗余，验证成本仅为Delta Debugging的一半。
   - 结果：在多个代理框架上，TRIM将CodeSlop减少了17.9%-32.9%，且性能回归可忽略。验证成本仅为Delta Debugging的一半，效率极高，表明轨迹最小化是一种有效的间接降冗余策略。

2. [LLM-as-a-Coach: Experiential Learning for Non-Verifiable Tasks](https://arxiv.org/abs/2607.18110v1)
   - 分类：LLM Training / Reinforcement Learning
   - 研究价值：值得精读
   - 关联：提出新的后训练范式，可应用于Agent行为优化
   - 痛点：强化学习在开放任务中依赖标量奖励，将评估压缩为一个数值，丢弃了丰富的文本反馈，并模糊了不同质量分布的响应。这导致监督信号带宽低，容易引发奖励黑客问题，且泛化能力受限。现有方法难以在非可验证任务中提供细粒度指导。
   - 方法：提出体验式学习（EL），将LLM-as-a-Judge转化为LLM-as-a-Coach，教练模型将每个响应的评估蒸馏为可传递的经验知识，通过上下文蒸馏让策略内化。与传统RL相比，提供了更高带宽的反馈通道，保留了细粒度偏好，并利用分布外泛化。
   - 结果：在多个策略家族和反馈来源下，EL持续优于基于规则标的RL，在分布外任务上泛化更好，并缓解了奖励黑客现象。这表明经验知识是一种更丰富、更可泛化的学习信号。

3. [FinSAgent: Corpus-Aligned Multi-Agent RAG Framework for Evidence-Grounded SEC Filing Question Answering](https://arxiv.org/abs/2607.18102v1)
   - 分类：Financial QA / Multi-Agent RAG
   - 研究价值：值得精读
   - 关联：多Agent RAG在金融领域的成功应用，方法可迁移
   - 痛点：现有RAG和多代理系统在SEC文件问答中，直接从用户问题推导检索查询并按语义相似度排序，导致先验-语料不一致：查询生成忽略文件的结构和术语，语义排序偏好主题相似但证据无效的段落。这种不一致降低了检索覆盖率和答案正确性，尤其在需要精确证据的长文文件中。
   - 方法：提出FinSAgent，一种语料对齐的多代理框架。它结合角色专用代理（锚定10-K结构）、数据库感知查询分解（基于语料摘要视图生成子查询）和多路径检索，并使用特征门控重排序器区分证据有效性和语义相似性。核心原则是注入语料侧条件来纠正模型先验主导。
   - 结果：在五个离线金融QA基准上，FinSAgent在检索覆盖率和答案正确性上优于单代理和多代理基线。在1000用户的随机在线实验中，它也获得更高的评分，表明其在实际场景中的有效性。

**主要痛点**
- Agent生成的代码存在冗余，导致代码库维护成本增加。
- 重排序器在生产部署中需要平衡效率、领域鲁棒性和半结构化数据支持。
- LLM在智能电网中可能输出物理不可行的结果，缺乏统一评估。
- 工业视频异常检测中VLM因对象变换和物理约束性能下降。
