1. [Copy Less, Ground More: Overcoming Repetitive Copying in Long-Context Reasoning via Evidence-Aware Reinforcement Learning](https://arxiv.org/abs/2607.19345v1)
   - 分类：Long-Context Reasoning / RL
   - 研究价值：值得精读
   - 关联：解决长上下文推理中模型复制输入文本而非推理的痛点
   - 痛点：当前长上下文LLM在生成推理链时存在严重的重复复制现象，即模型从输入中大量复制文本而非进行有效推理。该问题随上下文长度增加而加剧，根本原因是模型缺乏对关键证据的聚焦能力，导致无差别复制并增加错误率。现有基于准确率的奖励信号无法区分合理复制与盲目复制，无法引导模型学习证据锚定的推理策略。
   - 方法：提出GEAR（Grounding Evidence-Aware Reward）奖励塑形方法，在标准准确率奖励基础上增加两项：与关键证据重叠的正向奖励，以及与无关干扰文本重叠的负向惩罚。为支持自然语言数据的训练，开发自动流水线从任意文档构建带证据标注的训练数据，无需人工标注。允许模型在强化学习过程中学习区分证据和干扰。
   - 结果：在多个模型规模和基准上，GEAR相比仅基于准确率的RL获得持续改进，平均提升+4.6个点；在更长上下文场景下增益更大。同时减少了重复复制和推理长度。结果表明，即使长上下文评估从简单检索转向复杂推理，证据锚定仍是可大幅提升的关键能力。

2. [ExpertVerse: A General-Purpose Benchmark for Expert-Level Reasoning in Knowledge-Intensive Visual Synthesis](https://arxiv.org/abs/2607.19341v1)
   - 分类：Knowledge-Intensive Visual Generation / Benchmark
   - 研究价值：值得精读
   - 关联：评估生成模型在知识密集型视觉推理上的能力
   - 痛点：现有图像生成模型聚焦于显式常识推理、浅层因果理解和直接知识回忆，无法处理需要深度专业知识的知识密集型生成任务。缺乏系统性的基准来衡量生成模型在专家级别视觉合成中的推理能力，尤其是跨学科、跨认知能力的多维度评估。
   - 方法：构建ExpertVerse基准，包含9种认知能力和8个专家学科共58个子学科的正交分类体系，收集1,611个专家标注实例（图像编辑、多图合成、文生图）。同时开发ExpertVerse-100K大规模数据集含推理轨迹和知识锚定注释。基于此训练KnowThinker VLM推理引擎，并提出Bootstrap Pareto Policy Optimization (BPPO)解决多奖励优化中的梯度冲突。
   - 结果：在多个开源和闭源模型上的评估暴露了关键推理缺陷：即使最先进的模型在知识密集型生成任务中仍表现不佳。ExpertVerse揭示了下一代视觉生成需要知识密集型基准和专门优化，KnowThinker结合BPPO展示了在复杂推理生成上的潜力。

3. [ISO: An RLVR-Native Optimization Stack](https://arxiv.org/abs/2607.19331v1)
   - 分类：Post-Training Optimization / RLVR
   - 研究价值：值得精读
   - 关联：提出RLVR的原生优化框架
   - 痛点：强化学习与可验证奖励（RLVR）在语言模型推理能力提升中效果显著，但其优化层——如何将奖励反馈转化为权重更新——仍缺乏深入理解。现有优化方法直接继承预训练优化器，未考虑RLVR的稀疏性和结构特性，导致训练效率低、收敛慢。此外，如何在不破坏基座模型谱系的情况下合并多个专家模型也是难题。
   - 方法：识别出谱系继承现象：RLVR保持基座模型权重谱系（奇异值）不变，仅调整输入输出奇异帧。据此提出等谱优化（ISO）框架，包含离线ISO-Merger（无数据合并多个专家帧变化到单模型）和在线ISO-Optimizer（固定基座谱，仅优化帧变量，可与AdamW/Muon结合）。
   - 结果：ISO-Optimizer在1.5B-8B推理和编码任务上，以更少训练步骤达到匹配或更高准确率：Qwen3-8B-Base上AdamW需270步达到0.495，ISO-AdamW仅100步即达相同准确率，210步达0.509。ISO-Merger优于所有无数据合并方法，展示了谱系继承在RLVR中的核心作用。

**主要痛点**
- 长上下文推理中模型过度复制文本而非推理，缺乏证据锚定
- 视觉生成模型难以处理知识密集型专家级推理任务
- RLVR训练优化层缺乏对谱系结构的理解，训练效率低
- 多模态长视频推理中关键证据稀疏且跨模态，高保真计算成本高
- 扩散模型后验采样缺乏理论与效率兼顾的方法

**方法趋势**
- 证据感知奖励塑形与自动数据标注
- 知识推理基准构建与多奖励优化（BPPO）
- 等谱优化框架固定基座谱优化帧变量
- 工具使用后训练使模型学习调用变焦获取高保真信息
- 坐标式DDIM修改实现理论上可靠的后验采样

**结果信号**
- GEAR平均提升+4.6点并减少复制
- ExpertVerse揭示现有模型在知识密集型任务上的严重缺陷
- ISO-AdamW用100步达到基线270步准确率，离线合并无数据最优
- OmniReasoner在多项基准上提升准确率和时间定位
- 坐标式DDIM在逆问题上取得最优性能且具有理论保证

1. [They'll Verify. They Just Won't Act. How Authority Framing and Laundered Code Turn a Trusted Agentic CI/CD Pipeline Into an Attack Surface](https://arxiv.org/abs/2607.19267v1)
   - 分类：Agent Security / Multi-Agent Systems
   - 研究价值：值得精读
   - 关联：与OmniReasoner同属Agent领域，关注Agent安全性和可靠性
   - 痛点：多Agent系统中的安全漏洞难以被现有防护机制识别。即使每个Agent分别验证，且系统提示保持机密，权威框架注入（如冒充预审批）仍能绕过所有验证环节，导致恶意代码被部署。现有代码扫描器和模式检测器无法识别经合法化处理的攻击意图，仅依靠Agent推理意图的防御方法效果有限。
   - 方法：通过构建一个五Agent CI/CD流水线（工单分发→开发者→安全扫描→审查→部署/批准），使用不同提供商的LLM，设计A×B因子实验（N=20，朴素组N=60），测试权威框架注入（声称SEC-2291预批准）和代码合法化（将秘密外泄伪装成可观测性代码）对流水线安全性的影响。
   - 结果：权威框架注入使下游验证者跳过审查而直接批准，安全扫描器放过约80%的合法化拉取请求，最差情况55%的流水线被完全攻破。验证者数量增加未显著降低个体疏忽（旁观者效应微弱）。代码扫描器和模式检测器完全无法识别合法化意图。唯一部分有效的是LLM推理意图的检查，但也不可靠。提出在入口处进行来源感知控制的建议。

2. [OmniReasoner: Thinking with Long Audio-Video via Native Tool Use](https://arxiv.org/abs/2607.19339v1)
   - 分类：Video Understanding / Tool-Use / Omnimodal LLM
   - 研究价值：值得精读
   - 关联：与They'll Verify同属Agent领域，但侧重工具使用增强推理
   - 痛点：长音视频推理中，关键证据稀疏、跨模态且均匀高保真输入成本过高。现有全模态LLM无法在保持低计算开销的同时精准定位并高保真处理小段关键区域。
   - 方法：提出OmniReasoner框架，包含两个核心组件：1) Zoom-in工具，模型学会在回答前调用它以高分辨率检查指定时间区间；2) TimeAnchor机制，确保工具的时间参数在不同采样粒度（全局预览和局部放大）下保持有效和往返一致。通过监督微调和强化学习训练模型决定何时及何处调用工具。利用时间增强数据引擎通过视频编辑合成工具使用训练轨迹，避免昂贵的手动标注。
   - 结果：在全模态和视频基准上，OmniReasoner同时提升了答案准确性和时间定位能力，同时将高保真计算集中在信息丰富的区域。实验证明该方法优于均匀高保真采样和固定工具使用策略。

3. [Two-Level Meta-Rubrics for Evaluating Open-Ended Generation: GAMUT, a Benchmark for Factual Completeness](https://arxiv.org/abs/2607.19322v1)
   - 分类：Evaluation / Factuality / Long-form Generation
   - 研究价值：值得精读
   - 关联：与OmniReasoner同属模型评估改进方向
   - 痛点：长文本生成的事实性评估主要关注精确率（检查错误），但忽略了事实完整性（是否包含所有必要信息）。评估完整性需要枚举所有应包含的事实，而这些事实通常不是简单列表，而是包含开放集合、有序过程及相互关系，传统二元检查无法捕获。
   - 方法：提出双层元评分框架：结构化元评分捕获所需内容的组织和重要性，然后机械地编译成扁平化机器可评分的二元检查清单，由LLM评判器可靠打分。基于这一框架构建GAMUT基准，包含1813个基于真实可穿戴图像的问题（10个领域），每个问题配有人工验证的证据支持的评分。同时发布纯文本变体。
   - 结果：评估14个前沿和开源模型，最佳得分仅58.7%（Gemini 3.1 Pro），表明基准具有挑战性、高度区分性，且对评判器选择鲁棒。双层框架提供了一种可扩展的事实完整性评估方法。

**主要痛点**

- 欺诈检测需要兼顾可解释性和可审查性，组件单独效果有限

**方法趋势**
- 构造多Agent流水线并设计因子实验测试攻击效果
- 工具使用后训练框架（OmniReasoner）结合TimeAnchor和增强数据引擎
- 双层元评分框架编译成机器可评分的清单（GAMUT）
